## Phase 1 · 온라인(Firebase) 최적화 — 4건 수정

| # | 문제 | 수정 |
|---|---|---|
| 1 | **입력을 변화가 없어도 초당 30회(1v1)/20회(팀) 재전송** — 가만히 있어도 학생 1명당 초당 20~30회 DB 쓰기. 학급 동시 플레이 시 Firebase 쓰기 한도(초당 ~1000회)를 낭비하는 최대 요인 | 입력은 **변할 때 즉시** 전송(반응성 동일) + 변화 없으면 0.75초 keepalive만. 유휴 시 쓰기 **30/초 → 1.3/초 (약 96%↓)** |
| 2 | **호스트가 탭을 닫으면 방이 DB에 영구 잔류** — 정리(remove)가 `setTimeout 800ms`라 페이지가 먼저 죽어 실행 불가. 방 데이터가 무한히 쌓임 | 나가기 시 `onDisconnect().remove()`를 함께 무장 — 연결이 끊기는 순간 서버가 방을 청소(정상 퇴장 시엔 no-op) |
| 3 | **종료된(ended) 1v1 방에 입장 가능** — 유령 방에 들어가 무한 대기 | 입장 시 status="ended" 거부("이미 종료된 방이에요") |
| 4 | **방 코드 충돌 시 죽은 방을 재활용 못 함** | 방 생성 시 meta 없음/ended 방은 빈 코드로 간주하고 덮어씀 |

유지 결정(의도적): 호스트 상태 동기화 30Hz(1v1)/15Hz(팀)는 조작감 직결이라 유지. 호스트 순간 접속 끊김 시 방을 즉시 삭제하지 않고 기존 status=ended 동작 유지(재시작 가능성 보존 — 교실 와이파이 블립 대비).

## Phase 2 · 렌더/프레임 성능 — 8건 수정 (크롬북 프레임 방어)

1. **파이터 글로우 `createRadialGradient` 매 프레임 생성 → 색·반경별 캐시** (도형 fallback·보스 크롬 — 개체당 프레임당 1회씩 만들던 것 제거)
2. **아이템(하트 등) 글로우 그라디언트 → 스타일별 1회 생성**
3. **블랙홀 장판 그라디언트 → 원점 translate + 반경별 캐시**
4. **`shadowBlur` 제거(캔버스에서 가장 비싼 연산)**: 타이틀 "별빛 아레나"(매 프레임 60px 블러)와 게임 중 "보스 등장!"(1.6초간 매 프레임 70px 블러)을 **오프스크린에 1회 구운 글로우 스프라이트**로 교체 — 펄스 연출 동일(글로우 층 알파만 변조). 하니스(Node)에서는 자동으로 기존 경로 fallback
5. **탄환 명중 판정의 타겟 배열을 서브스텝 루프 밖으로** (탄환×스텝마다 새 배열을 만들던 GC 압박 제거) + 제곱거리 비교로 `Math.hypot` 제거
6. **`worldImage` 경로 문자열 사전 조립** (바닥/함정/구조물 렌더가 매 프레임 수십 번 문자열 3개를 이어 붙이던 것 제거)
7. **HUD 매치 정보 줄 캐시** (매 프레임 선형 검색 4회+문자열 6조각 조립 → MATCH 바뀔 때 1회) · **save/restore를 탄환·효과마다 → 함수당 1회** · drawFighter 시점 객체 재사용 · update() 파이터 루프 병합 · 버프오브 루프의 배열 생성 호이스트

## 후속 권고 (이번에 안 고친 것 — 사유 포함)

1. **1v1 상태 패킷의 탄환 색상 필드(c) 생략 가능**(~10B×40발×30Hz): 무기 id로 역산 가능하나 스킬탄 예외 처리가 필요해 이득 대비 회귀 위험으로 보류.
2. **탄환/효과 배열의 매 프레임 filter 재생성 → in-place 압축**: 이득 소폭, 코드 복잡도 증가로 보류.
3. **버프오브 TTL 없음**(구석에 떨어진 오브가 매치 끝까지 잔존): 성능 영향 미미하고 TTL은 밸런스 변경이라 수업 튜닝 때 결정 권장.
4. **결과/일시정지 화면의 shadowBlur**: 게임플레이 화면이 아니라 프레임 영향 없음 — 보류.
5. 탄환 시뮬 루프 4벌 복붙 구조: 공통 헬퍼 추출은 대규모 리팩터링이라 별도 트랙 권장(이번엔 4벌의 드리프트만 동기화).
