# 로비개편 결과 보고 (LOBBY-REDESIGN-1) — 2026-07-05

시작 화면(로비) 정보구조를 3층으로 개편하고 붐빔을 제거한 작업의 결과 문서.
스펙: `로비개편_spec.md` · 버전: v1.26

---

## 1. 개편 배경

- 기존 로비는 중앙 메뉴 9행 평면 나열(바로시작·온라인·캐릭터·캠페인·난이도·맵·대전방식·로그아웃·교사용)로,
  "혼자/함께"라는 첫 질문이 읽히지 않고, 설정 3종이 어느 모드에 적용되는지 불명확했다.
- 배경이 실제 게임 맵(가시·바위·격자)을 62% 딤으로 렌더해 정체불명의 어수선함을 만들었다.
- 좌측 "내 정보" 패널이 상단 식별 바와 정보 중복(이름·레벨·골드), "더보기 · 상점 준비 중" 등 공사판 문구 노출.

## 2. 3층 IA 결과

```
①껍데기(상단바):  [👤 이름·Lv·ID] [💰 골드] (U 프로필)      ● 온라인상태 [⚙ 설정]
②섹션 스트립:     [🏠 홈(현재)] [🛒 상점] [📖 수집]
③플레이(중앙):    혼자 하기
                    ⭐ 캠페인 도전        20스테이지 몬스터전
                      └ 난이도
                    ▶ 빠른 대전           단판
                      └ 규칙 · 팀 · 맵 · 난이도
                  친구와
                    🌐 온라인 센터        방 만들기 · 참가
우측: ⭐ 내 캐릭터 카드(C 캐릭터 변경) — 유지
```

- **설정 스코프(관통 규칙)**: 캠페인 밑에는 난이도만, 빠른 대전 밑에는 규칙·팀·맵·난이도.
  두 난이도 행은 같은 값(`selectedDifficultyId`)을 공유한다(소비 배선 불변).
- **⚙ 설정 오버레이**(Esc 또는 설정 칩): 별명 변경 · 로그아웃 · 교사용 모드.
  새 STATE 없이 `settingsOpen` 불리언 오버레이(=confirmLeave 방식)로 구현.
- **정리**: 2대1(duo) 모드 제거 · 중앙 캐릭터 행 제거(우측 카드+C 키) · 좌측 "내 정보" 패널 제거
  (이름·레벨·골드는 상단바 칩, 전적·수집은 U 프로필 화면 담당) · "상점 준비 중" 헤더 문구 제거.
- **규칙 셀렉터 자리**: `RULES`(현재 섬멸전 1종)·`selectedRuleId`·`cycleRule()` 신설.
  점령전·색칠전 게임플레이는 후속 spec(자리만 심음, 프로필 저장 없음).

## 3. 온라인 센터 수정

- 제목 "온라인 대전 (베타)" → "온라인 센터".
- 전역 맵 셀렉터(상단 "◀ 맵 ▶ ←→/M 변경 · PVE는 스테이지 맵" 라인) 제거.
- 그룹핑: **온라인 대전 — 친구와 PVP**(1대1·3대3 만들기 + 스코프된 맵 행) /
  **온라인 협동 — 함께 몬스터전**(PVE 만들기, 맵 라인 없음) / **참가**(방 코드 입장·뒤로가기).
- 맵 변경은 맵 행에서만 ←→/Enter(다른 행에선 무동작). 전역 KeyM 단축은 제거.
- "PVE는 스테이지 맵" 변명 힌트 제거.

## 4. 배경 정돈

- `render()`에서 `drawStaticWorld()`(맵 바닥·함정·풀숲·장애물)를
  PLAYING·PAUSED·OVER·온라인 4상태에서만 호출하도록 게이팅.
- 나머지 메뉴 화면(로비·스플래시·로그인·상점·수집·프로필·교사용)은 신규 `drawMenuStarfield()`
  (전체 화면 결정적 별밤 + 큰 별 3개 트윙클)만. 로비의 `dimScreen(0.62)` 제거.
- 플레이 화면 맵 렌더는 무변경(브라우저 육안 확인 — 바닥 타일·수풀·수정·HUD 정상).

## 5. 인덱스 재매핑 내역

| 항목 | 기존(v1.20~25) | 신규(v1.26) |
|---|---|---|
| MENU_ROWS | 9 | 11 |
| VALUE_ROWS | {2,4,5,6} | {1,3,4,5,6} |
| row0 | 바로 시작 → startGame | ⭐ 캠페인 → startSoloCampaign |
| row1 | 온라인 센터 | └ 난이도(값) |
| row2 | 캐릭터(값) | ▶ 빠른 대전 → startGame |
| row3 | 캠페인 | └ 규칙(값 · cycleRule) |
| row4 | 난이도(값) | └ 팀(값 · cycleMode) |
| row5 | 맵(값) | └ 맵(값 · cycleMap) |
| row6 | 대전방식(값) | └ 난이도(값) |
| row7 | 로그아웃 | 🌐 온라인 센터 → openOnlineMenu |
| row8 | 교사용 | 🛒 상점 → openShop (비활성 시 토스트) |
| row9 | — | 📖 수집 → STATE.COLLECTION |
| row10 | — | ⚙ 설정 → settingsOpen 오버레이 |

- 8~10행(섹션·설정)은 ←→로도 가로 이동. 단축키 유지: C 캐릭터·D 난이도·M 맵·T 팀·N 별명·O 온라인·U 프로필·I 수집·Q 로그아웃, Esc=설정.
- 설정 오버레이 내부: 0 별명 변경 · 1 로그아웃 · 2 교사용 모드(ADMIN_LOGIN).
- `goToStartScreen()`이 `settingsOpen=false`로 오버레이 잔류를 방지.

## 6. 테스트 결과

- **하니스 9종 ALL PASS**: harness · 3v3 · abilities · accounts · checkpoint · fixed_loadout · online · pve · studentmaps
  (sim_pve는 리포터 — exit 0).
- 하니스 갱신분: harness.js(로비 매핑 15장을 새 구조로 재작성 · duo 플레이 테스트 제거),

  맵 행 ←→ 동작·비맵 행 무동작 확인, 빠른 대전 진입 시 플레이 화면 맵 렌더 정상.
  콘솔 JS 에러 0(몬스터 PNG 404는 기존 fallback 설계·favicon 404로 개편과 무관).
