# 별빛 아레나 — 학생 에셋 교체 가이드 (교사용)

> 📌 **v1.14 업데이트**: 1차 적용은 방향별 모션이 아니라 **idle/move/attack/hit/return 5프레임** 방식으로 바뀌었습니다.
> 실제 적용 절차는 **`STUDENT_ASSET_INTAKE.md`** + **`tools/slice_assets.py`** 를 보세요(reference 시트 3장을 자동으로 잘라 넣습니다).
> 캐릭터-도구는 **고정 매칭**(럭키↔별조각, 달이↔종이비행기, 시고니↔반짝부메랑, 눈꽃↔비눗방울, 모아↔쿠션방패, 별골렘↔젤리캡슐).
> 아래 문서는 폴더/파일 규칙의 일반 설명입니다(파일 이름 규칙만 5프레임으로 이해하세요).

학생들이 직접 그린 **캐릭터 6개**와 **놀이도구 6개**를 게임에 넣는 방법입니다.
**PNG가 없어도 게임이 그대로 돌아갑니다(도형 fallback).** PNG가 준비되면 아래 순서대로 넣으면 됩니다.

---

## 1. 전체 흐름 (한눈에)

1. 학생 그림(사진/스캔)을 모은다.
2. 그림을 바탕으로 AI(또는 직접)로 **투명 배경 PNG**를 만든다. (프롬프트 예시는 7번)
3. PNG를 정해진 **폴더/파일 이름**으로 저장한다. (2·3번 규칙)
4. `index.html` 맨 위의 `let ASSETS_ENABLED = false;` 를 **`true`** 로 바꾼다.
5. 게임을 열고 시작 화면에서 **F2**(에셋 확인)로 어떤 그림이 연결됐는지 확인한다.
6. 잘 나오면 끝. 안 나오는 칸은 “fallback 사용 중”으로 표시되고 도형으로 보인다.

> 파일이 없거나 이름이 틀리면 그 부분만 자동으로 **도형(fallback)** 으로 나옵니다. 게임은 절대 멈추지 않습니다.

---

## 2. 캐릭터 파일 이름 규칙

폴더: `assets/characters/<assetId>/` (assetId 는 `student_01` ~ `student_06`)

| 파일 이름 | 뜻 |
|---|---|
| `down_idle.png` `down_walk1.png` `down_walk2.png` | 앞모습(서기/걷기1/걷기2) |
| `up_idle.png` `up_walk1.png` `up_walk2.png` | 뒷모습 |
| `left_idle.png` `left_walk1.png` `left_walk2.png` | 왼쪽 |
| `right_idle.png` `right_walk1.png` `right_walk2.png` | 오른쪽 |
| `hit.png` | 맞는 모션(없으면 idle 사용) |
| `ko.png` | 탈락/리턴 모션(없으면 기존 효과) |
| `hold_down/up/left/right.png` | 도구를 든 모습(선택, 캐릭터 위에 덧그림) |

- 최소한 `down_idle.png` 하나만 있어도 그 방향은 그림으로 나옵니다.
- 걷기 모션이 없으면 자동으로 idle 한 장만 사용합니다.
- 권장 크기: **128×128 PNG, 투명 배경**.

## 3. 무기/도구 파일 이름 규칙

폴더: `assets/weapons/<assetId>/` (assetId 는 `tool_01` ~ `tool_06`)

| 파일 이름 | 뜻 |
|---|---|
| `icon.png` | 선택 화면용 아이콘(선택) |
| `projectile.png` | 발사체 그림(없으면 기존 원형 탄환) |
| `held_down/up/left/right.png` | 캐릭터가 든 모습(선택) |
| `effect_hit.png` | 적중 효과(선택, 추후 확장용) |

## 6. 어떤 부분을 바꾸면 되나 (index.html)

`index.html` 맨 위 **★ 학생 캐릭터 6슬롯 / 학생 도구 6슬롯 ★** 구역에서
`// STUDENT_EDIT` 가 붙은 곳만 바꾸세요.

- 캐릭터: `name`(이름), `desc`(설명), `superName`(특별기술 이름), `color`/`subColor`(대표색)
- 도구: `name`, `desc`, `color`
- **능력치(hpMul, speedMul, damageMul …)는 건드리지 마세요.** 균형이 이미 맞춰져 있습니다.
- `assetId` 는 그대로 두는 것을 권장(그래야 폴더 이름과 자동으로 맞음).

그림을 켜는 스위치:
```js
let ASSETS_ENABLED = false;  // → true 로 바꾸면 PNG 사용
```

## 7. 학생 그림 → AI 에셋 프롬프트 예시

> 이 학생 그림을 바탕으로 탑다운 2D 아레나 게임 캐릭터 에셋을 만들어줘.
> 귀엽고 캐주얼한 초등학생용 스타일, 굵은 외곽선, 밝은 색감, 투명 배경, 128×128 PNG,
> 앞/뒤/왼쪽/오른쪽, 걷기 모션 2장, 폭력적인 무기 없이 놀이도구 느낌.

도구(놀이도구) 프롬프트 예시:

> 이 학생 그림을 바탕으로 탑다운 2D 게임의 놀이도구 아이콘과 발사체를 만들어줘.
> 투명 배경 PNG, 굵은 외곽선, 밝은 색, 총·칼·폭탄 같은 무기 느낌이 아니라
> 별조각·비눗방울·종이비행기·젤리 같은 놀이도구 느낌으로.

## 8. 주의 — 따라 하지 않기

- **유명 게임/애니메이션 캐릭터를 그대로 따라 그리지 않습니다.** (마리오, 포켓몬 등 X)
- 학생 본인의 창작 디자인으로 만듭니다.

## 9. 주의 — 폭력적 무기 대신 놀이도구

- 총·칼·폭탄 같은 명칭/그림 대신 **놀이도구 느낌**으로 만듭니다.
- 화면에 쓰는 말도 “도구 / 발사 / 에너지 / 리턴 / 특별기술” 중심으로 유지합니다.

## 10. PNG가 없을 때 fallback이 보이는 이유

- 그림이 없거나 이름이 틀리면, 게임이 멈추지 않도록 **원래의 도형 캐릭터/원형 탄환**을 대신 그립니다.
- 이것은 오류가 아니라 정상 동작입니다. F2 화면에서 “fallback 사용 중”으로 표시됩니다.

## 11. 나중에 Claude Code에 줄 “학생 에셋 반영 프롬프트” 예시

> 별빛 아레나에 학생 에셋을 반영해줘.
> `assets/characters/student_01 ~ student_06`, `assets/weapons/tool_01 ~ tool_06` 에
> PNG를 넣었어(파일 이름은 ASSET_GUIDE.md 규칙대로).
> `ASSETS_ENABLED` 를 true 로 바꾸고, index.html / star_arena.html 동기화한 뒤
> 4개 harness 테스트를 돌리고, F2 에셋 확인 화면에서 6슬롯이 모두 연결됐는지 점검해줘.
> 게임 기능·온라인·밸런스는 바꾸지 말고, 누락된 그림은 fallback으로 두면 돼.
