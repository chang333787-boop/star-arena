## §0. 배경·목표·비목표

### 0.1 배경
'AI 게임 만들기 수업'의 다음 단계. 공책게임을 넘어, 교사가 챕터 1의 기초 틀을 게임으로 구현하고 **학생들이 'RPG 위키'를 공책게임처럼 채워가며 세계관을 확장**한다. 모든 콘텐츠는 학생이 표 하나를 채우면 게임에 반영되는 **데이터 주도 구조**여야 한다.

### 0.2 성공 기준
1. 시고니 단독 주인공이 **맨손(→호미) 레벨 1**에서 시작해 챕터 1 보스를 잡고 **반짝 부메랑(아레나 시그니처)을 완성**한다. 완주 분량: **빠른 학생 90분 / 표준 5차시**.
2. 마을(NPC 6·상점 2·밭)·필드 2·던전 1이 돌아가고, 농사와 전투 루프가 퀘스트로 3회 강제 교차한다(§3.1 교차점).
3. 콘텐츠 10종 테이블(§7.1) 전부 `RPG_DB` 1행 = 콘텐츠 1개로 등록되고, **학생 종이 시트 → JSON → 교사 승인 → 게임 반영** 파이프라인이 E2E로 동작한다. **기본 콘텐츠 전량이 자기 검증기를 통과한다(§7.5 채점표 — 하니스 검증)**.
4. 기존 모드 전종 하니스 무회귀 + `harness_rpg.js` 신설 PASS.
5. 교사 게이트: P1(ADMIN 전용)으로 출시, P2 학급 개방은 후속 Phase(§10.1 RPG-5).

### 0.3 비목표 (챕터 1에서 안 하는 것)
- 스크롤(추적) 카메라 — 화면 단위 맵 + 문 워프(§2). 챕터 2+.
- **요리·화덕**(→ 챕터 2. mq07 도시락은 작물 직접 납품으로 대체) · 낚시·계절·날씨·호감도.
- 스킬 트리·2차 전직 — 1차 전직 2갈래 맛보기만.
- **학생 맵의 게임 반영** — 6차시 지역 시트는 종이 설계 + 챕터 2 예고로 운영(§8.3). RPG 맵 에디터도 챕터 2+.
- 온라인 멀티(필드 동시 접속) — 세이브 동기화만.
- 사운드(훅만) · 보안 강화(교사 PIN은 기존 "수업용 간이 보호" 수준 — 코드 주석 선언 유지).

### 0.4 대상·톤·난이도 헌장
초등 교실(태블릿+키보드). 순화 명칭: 노출 명칭은 **도구**(내부 테이블명 weapons — §7.1 매핑표). **아무도 죽지 않는다**: 플레이어·몬스터 공통 소멸 연출 = **별가루가 되어 하늘로 반짝이며 돌아감**, 먹구름단 단원은 "뿅!" 연기 도망. 난이도 헌장: **탄/타격 1회 피해 ≤ 내 HP의 10%**(같은 시점 권장 레벨 기준 — 다중 탄 동시명중은 회피 실패이지 한 방이 아님. §7.5 하니스가 전 몬스터×권장 레벨 조합을 검증).

## §1. 모드 등록·진입 게이트

### 1.1 상태머신 (플랫포머 선례 복제 — 통합 지점 6곳)
```
STATE.RPG:"rpg" 1개 추가 (내부 화면은 RPG.ui 서브상태 — field/inven/shop/farm/dialog/sleep)
통합 지점 체크리스트(누락 시 입력 누수·터치 불달):
 ① update(dt) 최상단 조기 리턴   ② render() 최상단 조기 리턴
 ③ handleKeyPress 디스패처 분기(※실측 함수명 — handleKeyDown 아님)
 ④ handleTouchStart  ⑤ handleTouchEnd  (플랫포머도 이 2곳에 분기 실재)
 ⑥ inPlayState()에 STATE.RPG 포함(범용 터치 스틱이 PLAYING/ONLINE_PLAYING만 통과시키는 게이트 — 미포함 시 §13 터치 AC 불가)
```
- 전역: `RPG_CONST`(튜닝 상수 — 전부 B콘솔 "RPG" 카테고리) · `const RPG_DB` · `let RPG=null`(이탈 시 null) — PF의 상수·데이터·런타임·저장 4분리. (RPG/RPG_DB/RPG_CONST 전역명 무충돌 실측 확인됨.)
- 기존 전역 비접촉 원칙, 단 **부메랑·탄 로직은 "그대로 재사용"이 아니라 `rpgBullets` 별도 배열로 패턴 이식(코드 복제)** — 아레나 updateBullets는 ARENA clamp·walls·team에 결합돼 있어 직접 호출 불가(규모 추정에 반영 §10.1).

### 1.2 교사 게이트
| 단계 | 노출 | 구현 |
|---|---|---|

## §2. 월드·맵 — 화면 단위 맵 + 문 워프

**판정**: 탑다운 스크롤 카메라는 유일한 미보유 기반기술 → 챕터 1은 화면 전환 방식으로 회피. BIG-BATCH-2 Phase 4(고정+줌아웃 ≤48×22)의 좌표 변환 유틸을 재사용하되, **RPG 자체 렌더 경로의 타일 렌더러·터치 좌표 변환은 신규 ~150줄**(과대 광고 금지). **폴백**: Phase 4가 중단조건에 걸려 미확보 시, 36×16·48×22 맵을 22×10 화면 2~4개+문 워프로 분할(§2의 전환 메커니즘이 그대로 폴백이 됨 — RPG-1 블로킹 없음).

## §3. 스토리 챕터 1 — "흐려진 별빛"

**로그라인**: 별빛으로 농사짓는 별마중 마을의 별빛이 흐려진다. 할머니의 밭을 물려받아 이사 온 외계 고양이 시고니가, 별빛을 훔쳐가는 **먹구름단**을 호미 한 자루부터 막아내고, 챕터 끝에서 상징 도구 **반짝 부메랑**을 완성한다(아레나 시고니의 기원담).

### 3.1 메인 퀘스트 7비트 (보상 열 = 경제·경험치 산수의 단일 출처)
| # | 제목 | 사건·성장 | 농사 연결 | 보상(exp/G/기타) | 남는 '왜?'(수업 발문) |
|---|---|---|---|---|---|
| mq01 | 별마중 마을 도착 | 농장 상속·이동/대화 튜토리얼. 맨손→**호미**(촌장 선물) | 잡초 정리·첫 파종 | 15exp · 별무 씨앗×5 | 할머니는 어디로, 왜 떠났나? |
| mq02 | 밤손님 | 그림자 젤리의 밭 습격 → 호미로 첫 전투 (Lv1→2) | **물주기→취침→첫 수확 = 완료 조건** | 40exp · 30G | 온순하던 젤리들이 왜 내려오나? |
| mq03 | 별조각 도둑 | 부슬이들이 별조각+별씨앗 절도. 창고에서 **낡은 부메랑** 발견 (Lv3) | 별씨앗 도난(작물 잠금) | 60exp · 40G | 훔친 별빛을 어디에 쓰나? |
| mq04 | 마을의 부탁 | 의뢰 3건(배달·수확·퇴치) → **대장간 개방**·스킬 [빙글 베기] (Lv4~5) | 의뢰 1건은 작물 납품(**교차점 1**) | 80exp · 60G | 먼별은 먹구름단을 어떻게 잘 아나? |

| mq06 | 별광맥 방어전 | 야습 방어전 — **낮에 심은 빛나무 묘목 수만큼 방어 조명 생성(교차점 3)**. 완료 후 밭 확장 구매 개방 (Lv7) | 낮의 밭이 밤의 전장 | 130exp · 100G · **별조각×3(부메랑 재료 — 순환 의존 해소: 보스 이전 지급)** | 왜 훔치기만 하나? |
| mq07 | 먹구름 요새 — 뭉게대왕 | 진입 조건: **원정 도시락 = 별방울 콩×3 납품(교차점 2)**. 직전 대장간에서 **반짝 부메랑 제작**(낡은 부메랑+별조각×3, **강화 단계 승계**). 알 발견 → 보스 격파 → 축제 (Lv9~10) | 보스 게이트가 밭에서 시작 | 175exp · 150G · 칭호 "별빛 수호자" | "우리도… 빛이 필요했을 뿐이야." 알 속엔? |

메인 exp 600 + 사이드 130 = **퀘스트 730exp** / 사냥 ≈ 339exp ≈ **40~55킬**(§4.2와 정합). 메인 골드 460 + 사이드 120.

### 3.2 먹구름단
물건만 훔친다(사람 무해). 훔친 별빛은 요새의 '알'에게 — 이유는 챕터 1 미공개(최대 여백). 계급: 그분(?) → 뭉게대왕 → **날씨 간부단(학생 확장 슬롯)** → 부슬이(양동이 투구·2인 1조 대사 반쪽씩). 간부 시트=스키마: `{name, weather, catchphrase(유머 필수), minion_palette(기존 몬스터 2~3), lair, pattern(보스 기술 재조합), steals, weakness(웃긴 약점=전투 힌트), last_words}`. 기본 1명: 안개 간부 **보슬보슬**("앞이 안 보이지? 나도 안 보여…").

### 3.3 NPC 6명 (기존 주인공 6인 이름과 충돌 금지 — 검증기 금칙 목록)
촌장 **온별**(퀘 허브·전설을 절반쯤 까먹음) · 잡화상 **모모**(수집광, 구매/판매) · 대장장이 **볼트**(고물 로봇, 도구에 몰래 별 스티커) · 씨앗 가게 **리프**(작물과 대화 주장) · 수상한 여행자 **먼별**(밤 출현·정체 공모=여백) · 꼬마 **피코**(관찰 노트 = **RPG 위키의 인게임 화신**).

### 3.4 열린 여백 7 (위키 '미해결 사건' 게시판과 1:1)
①요새의 알과 '그분' ②잠긴 지하실 ③찢어진 유품 지도 ④빛나지 않는 별조각 ⑤따라오는 젤리 1마리 ⑥우물 밑 노랫소리 ⑦할머니 일기의 찢어진 마지막 장. 규칙: 여백은 항상 흔적과 함께, 채택 시 데이터 행 교체("세계가 우리 반 답으로 바뀐다"), 채워지면 새 여백 보충.

## §4. 전투·성장 코어

### 4.1 스탯·공식
최종 피해 = (도구 피해 + ATK) × 스킬배수 × 패시브 − DEF (최소 1). 비전투 3초 후 8/s 회복. 쓰러지면 별우물 부활·페널티 0. 시작 HP 60 · ATK 2 · SPD 233.

### 4.2 레벨 곡선 (Lv1→10)

### 7.4 검증 파이프라인 `validateRpgContent` (반려 없는 자동 조정 우선)
①필수·타입(명세 테이블 = 상한표와 단일 출처, **참조 타입 컬럼으로 id 담는 전 필드 열거** — drops·shop·reward·grantSkills·acquire.value·requires·goal.target/item/to·portals.to 전부) → ②**자동 조정 통합**: 범위 클램프 + 파워버짓 초과 시 **비율 스케일다운**(반려 대신) + 조정 내역 안내("공격력 9999→13로 조정했어요") — §8.4 '탈락 없음'과 정합, 클램프→반려 이중 메시지 제거 → ③금칙어(RPG_BANNED 신설 — 현행 필터 부재 실측 · 기존 주인공 6인명 포함 · **실명 검출 경고**) → ④참조 무결성(제출 시+부팅 병합 시 재검사, 끊김은 드랍+교사 경고) → ⑤플레이 가능성(map: flood-fill + **portals 목적지 통행 가능 + 역방향 포탈 존재 + 전 맵 연결 그래프 도달성**) → ⑥로드 시 재클램프. 최종 게이트 = 교사 승인 패널(스탯 막대+버짓 게이지 미리보기).

### 7.5 파워버짓·변환표 (기본 콘텐츠 전량 자기 통과 — 채점표 하니스 고정)
- **monster**: `hp/10 + atk×2 + speed/30 ≤ 20 + 티어×15` (티어: 마을0/들판1/숲길2/요새3 · 가변 이속은 시간가중 평균 · **isBoss=면제, 교사 직접 검수**). 채점: 젤리 12.8 / 뿔깡총 16.9 / 총사 12.2 / 가시딱지 22.3 / 부슬이 24.8 / 부엉이 20.6 / 그늘조각 31.3 — 전원 자기 티어 상한 내 ✓.
- **weapon**: `편도 DPS(dmg/cd) ≤ 15 + levelReq×3`(부메랑은 편도 기준 — 왕복은 숙련 보상). 채점: 맨손 10≤18 / 호미 15≤18 / 낡은 부메랑 10≤24 / 단검 22.9≤24 / 별검 21.3≤30 / 별활 15.3≤33 / 반짝 16.7≤42 — 전원 ✓.
- **skill**: `dmgMul×100 / cd ≤ 60`(power=dmgMul×100 정의). 게이지·패시브형은 공식 비적용 — 교사 검수 전용.
- **crop**: `sellPrice ≤ seedPrice×3` + 순이익/일 8~14 밴드.
- **⭐→절대치 변환표(몬스터 — PRD 부록 확정)**: hp[12,25,45,80,130] · atk[2,4,6,9,13] · spd[40,70,110,160,220] (⭐1~5). **⭐예산 7 내 전 조합의 버짓 최대값 = 28.5(1/5/1) ≤ 티어1 상한 35 — 합법 시트는 전원 자동 통과**(전수 검산을 RPG-0 게이트·harness_rpg 테스트로 고정 → "시트 규칙을 지켰는데 반려"라는 좌절 원천 차단). 도구·스킬 변환표는 같은 역산 원칙으로 RPG-0 산출물.

## §8. RPG 위키·교육 프레임워크

### 8.1 3층 구조
[1층 실물] 시트(1인 1장)+교실 게시판(털실 연결선) → [2층 정본] wiki/ 폴더(항목 md + rpg_db.json + 변환표_밸런스.md) → [3층 무대] 게임 내 '별의 도감'(피코의 관찰 노트). 단일 ID(M-07)가 시트→게시판→위키→json→도감 번호로 관통 — 효능감의 핵심 장치. 도감 v1은 **3탭(몬스터/도구/작물)**, 8탭 완성은 RPG-5. 카드 하단 크레딧 상시. 검토중=실루엣 "??? 연구 중". 빈 도감 번호 = 다음 설계 유인.

### 8.2 시트 8종 공통 원칙
공통 헤더(ID·이름·만든 사람·상태 도장) · **숫자 금지, ⭐색칠·보기 동그라미만** · ⭐예산제(트레이드오프 교육+밸런스 1차 방어 — §7.5와 역산 정합 보증) · 🔗연결 칸 필수(**저학년 1개·고학년 3개**) · 그림 칸("배경은 그리지 않아요"). 퀘스트 시트는 대사 2종을 **빈칸 채우기 템플릿**으로, nextHint는 선택 항목(25분 상한 준수). 지역 시트는 24×12 테두리·legend 사전 인쇄 양식.

### 8.3 8차시 수업 흐름 (**P2 학급 개방 후 운영이 기본** — P1 기간은 차시 도입 10분 공용 월드 시연+취침 교대로 대체)
1 세계관 탐험(개인 플레이·여백 선언) → 2 몬스터(M카드 20~30장 = 연결 원료) → 3 장비(**친구 몬스터의 재료로만** — 연결 강제) → 4 NPC/작물(마을 배치도) → 5 퀘스트(**털실로 카드 3개 잇기**) → 6 지역(모둠 종이 설계 — 게임 반영은 챕터2 예고) → 7 편집 회의(체크리스트 상호검토+스티커 투표) → 8 **업데이트의 날** 🎉(패치노트 낭독·자기 콘텐츠와 플레이). 각 차시 도입 10분은 개인 플레이 트랙(밭 관리·퀘스트 진행) — 작물 연속성 확보.
- **전사(시트→JSON)는 차시별 롤링**(7차시 후 일괄 금지 — 학급 100장+ 일괄 시 교사 5~10시간 실측 추산). RPG-4 산출물에 **배치 intake 도구**(사진 일괄→AI 변환→오류 목록) 포함. 조기 완성자용 연결 미션 카드.

### 8.4 거버넌스
**위키 수록은 관대하게, 게임 반영은 엄격하게** — 탈락 없음, 보류는 '아이디어 온실'. **원안 보존, 숫자만 조정**(조정 내역은 패치노트 투명 공개 = 밸런스 수업 소재). 가이드라인: 이름 한글 2~8자·실존 인물/급우 실명/기존 주인공 6인 금지 · 아무도 죽지 않는다(별가루) · "1학년 동생 기준" · 친구 항목 부정 사용은 동의 필수. **개인정보 안전 기본값: 공개 산출물(T3 커밋·클라우드 미러)은 별명/이니셜 강제, 실명은 교실 실물 게시판 한정, 승인 패널에 실명 검출 경고**. 크레딧: 도감 상시+패치노트 호명+칭호(3개 견습 기록가/7개 도감 연구원/채택 개척자).

### 8.5 운영 수명주기 (신설)
- **학년 교체**: 연도별 아카이브 — 전년도 학생 콘텐츠는 disabled 일괄 + 도감 '명예의 전당' 탭으로 이동(T1 영구 내장분 포함), 새 학급은 빈 위키+명예의 전당 열람으로 시작(계승/초기화는 교사 선택).
- **전입생·결석생**: 여백(§3.4)·빈 도감 번호를 우선 배정하는 보충 미션(연결 체인 이탈 방지 — "네가 채울 자리가 남아 있어").
