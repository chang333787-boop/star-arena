# 코덱스 온보딩 & 작업 브리핑 — 별빛 아레나 에셋 정비

> **이 문서를 먼저 통째로 읽고 시작할 것.** 너(코덱스)는 이 프로젝트를 처음 본다.
> 목적: 흩어진 에셋을 **맵 에디터를 만들기 전에** 하나의 규격으로 정비한다.
> 작성 맥락: 아래 진단·결정은 이미 사람과 합의된 것. 임의로 뒤집지 말고, 의문이 있으면 물어볼 것.

---

## 0. 프로젝트가 뭔가

- **별빛 아레나** — 크롬북 Chrome에서 인터넷 없이 도는 1대1 아레나 슈팅 게임. 순수 HTML/Canvas **단일 파일**(`index.html`, `star_arena.html`은 동일본).
- 현재 v1.25. 캐릭터 6·무기 6·몬스터/보스 13·월드요소 다수가 `assets/` 아래 PNG로 실제 적용 중(`ASSETS_ENABLED = true`).
- 다음 큰 목표 = **파일 내 맵 에디터**. 단, 그 전에 에셋 규격부터 잠근다.

## 2. 지금까지의 진단 (합의됨)

- **캐릭터**: 마스터 시트 기반 파이프라인이 있어 **화풍이 이미 상당히 통일**됨(셀셰이딩 치비). → **거의 손 안 댄다.**
- **몬스터·타일·장애물**: **마스터 시트가 없이 따로 생성**됨 → 캐릭터보다 광택·고대비 가챠톤으로 튀고, `boss_king`은 자리표시자 수준. **여기가 정비 대상.**
- **타일 구조 문제**: 수풀·벽 등이 "낱개 일러스트"라 격자에 여러 칸 깔면 지저분(경계 안 이어짐, 그림자 겹침). → **타일셋(9-슬라이스)으로 재구성** 필요.

## 3. 확정된 방향 (규격서 0장과 동일)

- **시점**: 3/4 오블리크(브롤스타즈식). 그리기 순서 **y-sort**(발밑 Y 기준).
- **일관성 앵커**: **캐릭터 우선.** 캐릭터 시트가 톤 기준 → **몬스터·배경을 캐릭터에 맞춤(채도·대비 톤다운)**.
- **그림자**: 그림에 굽지 않음 → **엔진이 발밑 공통 타원 그림자**를 그린다.
- **격자**: 1타일 = 56px, 아레나 22×10. footprint(막는 칸)를 오브젝트마다 명시.
- **지형(수풀·벽)**: 낱개 폐기 → 9-슬라이스 타일셋(가운데+가장자리4+모서리4).

## 4. ⚠️ 역할 경계 — 이거 어기지 마

- **너(코덱스)는 그림을 생성하지 않는다/못 한다.** 이미지 생성은 **사람이 ChatGPT 앱에서** 마스터 시트로 뽑아 repo에 넣어준다.
- 너의 일 = **슬라이스 · 배경제거 · 정규화 · 배선(ASSET_MANIFEST) · 타일셋 로직 · 맵 에디터 · QA 하니스.**
- 그림이 필요한 단계에서 임의로 이미지를 만들거나 API를 붙이려 하지 말고, **"사람이 시트를 넣어줄 때까지 대기"** 로 표시할 것.

## 5. 앞으로 할 일 (순서대로)

### ▶ Phase 0 — 지금 바로 (새 그림 없이 가능)
0-1. 위 1장 파일들 정독 + 코드의 에셋 로더/격자 지점 확인.
0-2. **현황 감사 리포트**(짧게): 현재 에셋이 어떻게 로드·배선되는지, footprint 데이터가 없다는 점, 슬라이서가 어떤 시트 레이아웃을 가정하는지 요약.
0-3. **에셋 QA 하니스** 하나 추가: 현재 전 에셋을 56px 격자에 깔아 육안 확인하는 페이지(기존 `harness_*.js` 패턴 참고). 정비 전/후 비교용.
0-4. `slice_assets.py`를 **새 "월드 마스터 시트"** 도 자를 수 있게 일반화할 지점 파악(레이아웃 파라미터화). 아직 실행 말고 **계획만** 리포트.

### ⏸ Phase 1 — 사람이 "월드 마스터 시트" 넣어준 뒤
1-1. 새 시트를 슬라이스 → `assets/monsters` 등에 반영(기존 파일 백업/교체).
1-2. `ASSET_MANIFEST`/`PVE_ENEMY_TYPES` 경로 정합성 확인, 깨진 참조 없게.
1-3. QA 하니스로 캐릭터 대비 채도·대비 위계 확인.

### ⏸ Phase 2 — 타일/구조
2-1. 수풀·벽 **9-슬라이스 타일셋** 렌더링(오토타일: 이웃 칸 보고 가장자리/모서리 선택).
2-2. **footprint 데이터 모델** 도입(오브젝트별 막는 칸), 충돌/배치에 사용.
2-3. 발밑 공통 **그림자 레이어** + **y-sort** 그리기 순서.

### ⏸ Phase 3 — 맵 에디터
3-1. 56px 격자 위 배치 에디터 골격(팔레트→칸 클릭 배치, footprint 단위, 저장/불러오기).
3-2. 규격서 기준으로 에디터가 에셋 규격을 강제하게.

## 6. 작업 규칙

- **한 번에 다 하지 말 것.** Phase 0부터, 각 단계 끝에 사람 확인.
- `index.html`/`star_arena.html`은 **바이트 동일** 유지(슬라이서 주석 참고 — ASSETS_ENABLED 이미 true면 불필요하게 건드리지 말 것).
- 캐릭터 에셋/마스터 시트는 사람 승인 없이 변경 금지("외형·비율·표정·무기 변경 금지" 규칙 유지).
- 큰 변경 전 무엇을 왜 바꾸는지 먼저 요약.

**너의 첫 작업 = Phase 0 (0-1 ~ 0-4).** 완료되면 감사 리포트 + 하니스 결과를 보고할 것. 그림이 필요한 Phase 1부터는 사람이 시트를 넣어줄 때까지 대기.
