━━━━━━━━━━━━━━━━ PHASE 1 · 오늘 에셋 전면 적용 + 격자 정렬 ━━━━━━━━━━━━━━━━
목표: 오늘 만든 에셋을 게임이 실제로 쓰게 + 타일/프롭이 56 격자에 맞게.
1-1 등록: ASSET_MANIFEST(및 로더)에 새 에셋 추가 — 바닥6·별광맥·별pickup·수정탑청홍·타일셋2. 함정 새버전 확실히 로드(캐시 무효화: 버전 쿼리/타이틀 버전 bump).
1-2 바닥 렌더 격자 정렬: 256px 고정 → **56의 배수(224px 권장)**로 렌더. 바닥이 격자·오브젝트와 맞물리게.
1-3 프롭 footprint 적용: 재작된 프롭(crystal/block/rock 1×1, monster_gate 2×1)을 `설계_footprint_모델.md`대로 — 밑면이 footprint 칸에 정렬, 렌더 크기 footprint 기반(112 소스→56 렌더 등). 충돌은 기존 사각형과 호환.
1-4 타일셋 오토타일: 수풀 bush_01 낱개 → bush_tileset 9-슬라이스 오토타일로 교체. 쿠션벽 cushion_wall → cushion_wall_tileset 오토타일. 이웃 보고 조각 선택(코너·엣지·센터).
1-5 바닥 테마 선택: 맵 데이터에 바닥 테마 필드(자갈/풀/눈/모래/나무/어둠) → 스테이지·맵별 적용 가능하게.
자체점검: 브라우저에서 바닥·프롭·수풀·벽이 격자에 맞고 새 그림으로 뜨는지. 함정 빨강/초록/하늘로 보이는지.

━━━━━━━━━━━━━━━━ PHASE 2 · 밸런스 정비 ━━━━━━━━━━━━━━━━
(게임플레이_통합_spec.md의 Phase1 값 기준, 아래는 오늘 추가 수정 — 이게 우선)
2-1 **핫존 → "점령전"으로 표시명 변경** (표절 회피). 내부 id는 유지 가능, UI 라벨만 점령전.
2-2 **눈꽃**: 정면 공격 부재로 너무 어려움 → **쓸 만한 정면(직선) 공격 부여**(예: 부채 3발 중 중앙 1발은 직선 관통 or 단발 직선 옵션). + **X(얼음 존)가 아레나 밖으로 펼쳐지는 버그 → 아레나 경계로 clamp**.
2-3 **별골렘**: 여전히 약함 → 확실히 강한 탱커로 추가 버프(예: HP 200→**260**, 무기 데미지 40→**52**, 방패 X 흡수량 상향, 필요시 상시 피해감소 10%). 체감될 만큼.
2-4 **모아**: X(회복 오오라)·궁(무적)이 **아군에게만** 적용되는지 확인·보장(적에게 이로운 효과 금지).
2-5 **달이**: 궁(관통 저격) 시전시간 거의 즉발 → **0.3~0.4초 시전(wind-up) 추가**.
2-6 **별모으기(stargrab)**: 매치 총시간 180초 → **60초**.
2-7 **포탑(siege)**: 별 수정탑 HP 1800 → **약 450(1/4)**. 포탑 자동공격은 유지.
자체점검: 각 캐릭 손맛/난이도, 눈꽃 정면 명중·X 경계, 별골렘 탱킹 체감, 모아 아군전용, 달이 시전감, 별모으기 60초, 탑 파괴 시간.

━━━━━━━━━━━━━━━━ PHASE 3 · 타일/요소 역할 검증 (효과 없으면 안 됨) ━━━━━━━━━━━━━━━━
각 요소가 "실제로 의미있게 작동"하는지 검토하고, 안 되면 고쳐서 다시 적용.
3-1 **수풀 은신**: 아군/나는 수풀 속 유닛이 보이고, **적은 수풀 속 유닛을 못 봄**(근접 시 드러남 규칙 유지). 팀별 가시성 제대로 되는지 검증·수정. 효과 미미하면 강화.
3-2 **함정**: 가시=데미지, 끈끈이=둔화, 얼음=미끌/가속 — 밟았을 때 실제 효과가 체감되게(약하면 강화).
3-3 **거점(점령전)**: 점유 게이지 실제 작동, 팀색 표시.
3-4 **포탑(수정부수기)**: 자동공격 실제 작동 + **새 수정탑 에셋(tower_blue/red) 적용**.
3-5 **별광맥/별(별모으기)**: 별 스폰·수집·드랍 작동.
자체점검: 각 요소를 맵에 놓고 실제 플레이로 효과 확인. "있으나 마나"면 수정.

━━━━━━━━━━━━━━━━ PHASE 4 · 맵 에디터 (교사 승인 흐름) ━━━━━━━━━━━━━━━━
목표: 학생이 모드별 맵을 만들고, 교사가 승인해 게임에 적용.
4-1 진입: 새 에디터 화면/상태. 맵 만들 때 **먼저 게임 모드 선택**(섬멸/점령전/수정부수기/별모으기).
4-2 모드별 팔레트: 선택 모드에 맞는 요소만 배치 가능. 공통(바닥테마·벽·수풀·함정·프롭·시작점) + 모드전용:
   - 점령전 → **거점(점령 공간)** 배치
   - 수정부수기 → 팀 수정탑 위치
   - 별모으기 → 별광맥 위치
4-3 배치: 56px 격자, footprint 기반 스냅. 바닥 테마 선택. 오토타일(수풀·벽) 반영.
4-4 **점령전 = 기본 좌우 대칭**(왼쪽 편집 → 오른쪽 자동 미러). 데이터는 `맵에디터_요소확정.md` 형식(왼쪽 절반 저장+미러) 활용.
4-5 **맵 크기 확장 대비**: 22×10 기본 + 더 큰 크기(예 36×16) 지원. 화면보다 크면 **고정+줌아웃**(스크롤 없이 다 담기). (추적 카메라는 후속.)
4-6 검증: 시작점 필수·연결성(플러드필)·대칭 등 로드 검증.
4-7 **교사 승인 흐름**: 학생이 만든 맵 → **관리자(교사)에게 제출**. 교사용 화면에서 받은 맵 목록 확인 → **적용할지 결정** → 승인 시 게임(맵 목록)에 반영되어 플레이 가능. (제출/저장은 로컬 또는 기존 계정 시스템 활용, 신규 서버기능 없이 가능한 범위로.)
자체점검: 모드별 맵 제작 → 제출 → 교사 승인 → 플레이 전 과정.
