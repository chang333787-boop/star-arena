## Phase 3 · 조작 3모드 — 완료 (키보드/마우스/터치 전부)

### 3-0 입력 추상화
- 의도 객체 `{ mx,my(아날로그 -1..1), aim(라디안|null), aimPoint, fire, special, ult, reload, pick }` — `getLocalIntent()`가 모드별 소스를 같은 형식으로 변환, 게임 로직(로컬 updatePlayer·게스트 예측·host 판정)은 의도만 소비.
- **온라인 패킷 확장**: `mvx/mvy`(0.1 양자화)·`aim`(0.05rad 양자화) 추가. host의 `applyIntentMoveAim()`이 아날로그 우선, 없으면 구형 불리언(up/down/left/right) — **하위호환**. 세 모드가 같은 패킷 형식 = 동기화 안전(스펙 요구).
- 설정(⚙) 첫 행 **"🎮 조작"** — ←→/Enter로 키보드/마우스/터치 순환, 프로필 저장.

### 3-2 마우스 모드 — 무빙샷(카이팅) 해결
- **WASD(+방향키 겸용) 이동 · 커서 조준 · 좌클릭 발사(홀드 연사) · 우클릭 X · Space C · R 장전**.
- 캔버스 스케일 역산(`canvasToLogical`)으로 커서→월드 좌표. 플레이 중 OS 커서 숨기고 **크로스헤어** 렌더.
- **카이팅 자동 검증**: A(왼쪽 이동) + 커서 오른쪽 → x 감소하면서 facing≈0(오른쪽 조준) — 후퇴 사격 성립. 원격(호스트 판정) 동일 검증.

### 3-3 터치 모드 — 태블릿
- **좌측 터치 = 이동 스틱**(터치 지점 앵커·반경 60), **우측 터치 = 조준 스틱**(반경 70, 데드존 0.25) — 우 스틱 홀드 중 **조준+연사**(브롤식 변형: '놓으면 1발' 대신 홀드 연사 — 연사 무기 특성상 아이들에게 유리. 스펙과의 차이점으로 기록).
- **스킬독 탭**: X·C 원 탭=스킬, Z 원 탭=장전. PVE 강화 3택1도 **카드 탭**으로 선택 가능(키보드 1/2/3 없이).
- 멀티터치(이동+조준 동시) 지원, 가상 스틱 시각화(앵커 원+노브).
- 검증: 스틱 수학·조준각·데드존·탭 래치 하니스 자동검증. **실기기(태블릿) 검증은 필요** — 에뮬 한계로 기록.

### 3-4 모드별 안내

## 회귀 안전장치
- 키보드 모드 = 기본값, 기존 코드 경로와 **동일 동작**을 하니스로 고정(이동 방향=조준·Z 발사·X/C keydown 경로 유지 — 이중 발동 없음).
