# 별빛 아레나 — 타일 에셋 규격서 v0.1

> 목적: 맵 에디터를 올리기 **전에** 에셋의 규격을 잠근다. 이 문서가 ⑴다음 에셋 주문 ⑵맵 에디터 코드 둘 다의 단일 기준(single source)이다.
> 근거: 현재 코드값 — `ARENA = {x:24, y:70, w:1232, h:576}`, `STUDENT_CELL = 56`, `STUDENT_COLS=22`, `STUDENT_ROWS=10`.

---

## 0. 확정 결정 (이 문서의 전제)

| 결정 | 값 | 이유 |
|---|---|---|
| **시점(투영)** | 3/4 오블리크 (브롤스타즈식) | 기존 그림 대부분 활용·참고작과 일치 |
| **일관성 앵커** | 캐릭터 매력 우선 → **통일 스킨 = 깨끗한 셀셰이딩 치비**(브롤+쿠키런 톤) | 캐릭터가 주인공. 개성 유지 + 화풍만 통일 |
| **세계(몬스터·배경)** | 형태 유지 + **채도·대비 톤다운(조연화)** | 주인공을 띄우기 위해 배경은 뒤로 |
| **그림자** | 그림에 굽지 않음 → 엔진이 발밑 공통 타원 | 격자 위 그림자 충돌 제거 |

### 🔑 3/4의 생명줄 규칙 (전 에셋 공통)
1. **접지선(발밑 기준선)** 을 공유한다 — 캔버스 하단 일정 지점에 물체의 "바닥에 닿는 면"이 오게.
2. **광원은 좌상단 고정.** 하이라이트/그림자 방향 전부 동일.
3. **배경 투명(alpha), 자체 그림자·받침 금지.** 그림자는 엔진이 그린다.
4. 그리기 순서는 **y-sort**(발밑 Y가 큰 물체를 나중에 그림) → 앞뒤 겹침 자연스럽게.

## 1. 격자·좌표 규격

- **1 타일 = 56 × 56 px** (코드의 `STUDENT_CELL`)
- **아레나 = 22 × 10 타일** (원점 24,70)
- **작업 해상도(authoring)**: 표시의 **2배(@2x)로 그리고 축소** 권장 → 타일 소스는 112×112로 만들어 56으로 렌더 (크롬북 고DPI 대비, 선명도 확보)

### footprint = 오브젝트가 "막는 칸"
그림 크기와 별개로, **몇 칸을 차지/충돌하는가**를 반드시 명시한다. 맵 에디터는 이 footprint 단위로 배치·충돌 처리.

| 오브젝트 | footprint(칸) | 소스 캔버스(@2x) | 접지선 위치 |
|---|---|---|---|
| 캐릭터 | 1 × 1 (충돌원 r≈22) | 112 × 140 (세로 여유=키) | 하단에서 12px 위 |
| 몬스터(일반) | 1 × 1 | 112 × 128 | 하단 |
| 보스 | 2 × 2 | 224 × 240 | 하단 |
| 수정·바위(장애물) | 1 × 1 | 112 × 128 | 하단 |
| 몬스터 문(구조물) | 2 × 1 | 224 × 140 | 하단 |
| 벽(쿠션벽) | 1 × 1 **타일셋** | 112 × 112 조각들 | 하단 |
| 수풀(엄폐) | 1 × 1 **타일셋** | 112 × 112 조각들 | 바닥 평면 |
| 함정(가시·둔화·얼음) | 1 × 1 (바닥 데칼) | 112 × 112 | 바닥에 눕힘 |
| 회복 픽업 | 1 × 1 | 112 × 112 | 살짝 떠 있음(그림자만 바닥) |
| 바닥 타일 | 1 × 1 (이음 tileable) | 112 × 112 | — |

## 2. 통일 스킨 스타일 바이블 (셀셰이딩 치비)

전 캐릭터·몬스터가 **이 한 벌의 규칙**을 공유한다.

| 요소 | 규칙 |
|---|---|
| **비례** | 2 ~ 2.5 등신 치비. 머리 크게, 손발 단순화 |
| **라인** | 굵기 **일정**한 외곽선. 검정 금지 → **어두운 남보라(#2a2350 계열)** 컬러 라인. 안쪽 디테일 라인은 더 얇게 |
| **셰이딩** | **셀(2~3단)**. 그라데이션·에어브러시 금지. 광원 좌상단 → 우하단이 그림자 |
| **하이라이트** | 상단 가장자리에 밝은 셀 1단 + 눈/금속에만 작은 점 하이라이트 |
| **눈** | 큰 눈, **동일한 눈 스타일**(오타쿠 애니체 X, 브롤/쿠키런톤 O) |
| **채도 위계** | 캐릭터 = **고채도·고대비**(주인공) / 몬스터 = 중채도 / 지형·바닥 = **저채도**(뒤로) |
| **테두리 광(rim)** | 세계와 붙이기 위해 **별빛 골드 얇은 림** 허용(과하지 않게) |

### 공용 팔레트 (별빛 세계 기준)
| 역할 | HEX |
|---|---|
| 밤 배경 블루(바닥·조연) | `#2f47b0` / 딥 `#1b2a72` |
| 별빛 골드(공용 액센트) | `#ffd76a` |
| 하이라이트 화이트 | `#f4f8ff` |
| 시안 보조 | `#66e0ff` |
| 라인(검정 대체) | `#2a2350` |
> 캐릭터별 **고유색**(럭키=노랑, 눈꽃=파랑…)은 유지하되, 위 공용색을 하이라이트/그림자/림에 섞어 세계와 묶는다.

## 3. 카테고리별 처리 방침

### 3-1. 캐릭터 — 개성 앵커 유지 + 새 스킨 (3갈래 → 1갈래)
아래 **앵커(실루엣+핵심색+시그니처)** 는 반드시 유지, 라인·셰이딩·비례·눈은 스킨 규격으로 재드로우.

| ID | 이름 | 유지할 앵커 | 현재 화풍 → 목표 |
|---|---|---|---|
| student_01 | 럭키 | 노란 뭉게구름 블롭·눈 감은 미소 | 플랫 → 셀 치비 |
| student_02 | 달이 | 갈색 곰 + 파란 구슬 | 페인터리 → 셀 치비 (※실총→별 무기로 톤 정리 검토) |
| student_03 | 시고니 | 초록 외계 고양이·더듬이 | 플랫 → 셀 치비 |
| student_04 | 눈꽃 | 파란 여우 소녀 + 부채 | 애니 → 톤 순화한 셀 치비 |
| student_05 | 모아 | 숲 소녀 + 분홍 젤리 | 애니 → 톤 순화한 셀 치비 |
| student_06 | 별골렘 | 보라 별 로봇 | 페인터리 → 셀 치비(로봇도 치비 비례로) |

### 3-2. 몬스터 — 형태 유지 + 톤다운
- 9종 일반 + boss_robot/turtle/plane: **형태는 좋음.** 채도·대비만 낮춰 캐릭터에 주역 양보.
- **boss_king만 전면 재작** (현재 자리표시자 수준 → 다른 보스급 셀 치비로).

### 3-3. 지형(수풀·벽) — 낱개 일러스트 폐기 → **타일셋**
"여러 칸일 때 지저분" 문제의 정답. 하나의 그림이 아니라 **이어지는 조각 세트**로 재주문.

**최소 9-슬라이스 구성** (블롭 47까지 확장 가능):
```
[모서리_좌상] [가장자리_상] [모서리_우상]
[가장자리_좌] [ 가운데   ] [가장자리_우]
[모서리_좌하] [가장자리_하] [모서리_우하]
```
→ ㄱ자·ㄴ자·6칸 어떤 형태든 경계가 이어져 **하나의 매끈한 덩어리**로 보임.

### 3-4. 낱개 프롭(수정·바위·문·회복·함정)
- footprint·**접지선 통일**, 자체 그림자 제거.
- 같은 종류는 **변형 2~3종**(예: 바위 A/B/C)으로 복붙 티 제거.
- 함정은 **바닥에 눕힌 데칼**(높이 최소화)로 통일 → 밟는 칸임이 즉시 읽히게.

## 4. AI 재생성 프롬프트 템플릿

> 사용법: 아래 **[공용 스타일 블록]** 을 모든 프롬프트 끝에 그대로 붙인다. 이게 일관성의 핵심 장치. 기존 그림이 있으면 **레퍼런스 이미지(img2img)** 로 함께 넣어 실루엣을 유지.

### [공용 스타일 블록] — 전 에셋 공통 접미부
```
Style: clean cel-shaded chibi, 2 to 2.5 head proportion, thick even outline in dark
navy-purple (#2a2350) NOT black, 2-3 step flat cel shading (no gradients, no airbrush),
soft top-left light source, small crisp highlights, cute Brawl-Stars / Cookie-Run tone
(NOT anime otaku). 3/4 top-down oblique view. Palette anchored to a starry night world:
royal blue, star gold (#ffd76a), white highlights, cyan accents. Transparent background,
NO drop shadow, NO ground base/pedestal, object stands with feet at bottom baseline.
High saturation and contrast (this is a hero character). Centered, full body, game asset.
```

### 캐릭터 프롬프트
```
A {역할} chibi game character named "{이름}": {앵커 설명 — 실루엣·핵심색·시그니처}.
Keep this exact identity and silhouette. [공용 스타일 블록]
```
예(럭키):
```
A balanced-type chibi character "Lucky": a round fluffy yellow cloud-blob creature with
a happy closed-eye smile, tiny stubby feet. Keep this exact identity and silhouette.
Style: clean cel-shaded chibi ... (공용 스타일 블록)
```

### 몬스터 프롬프트 (톤다운)
```
A cute chibi monster: {형태 설명}. Slightly LOWER saturation and contrast than the hero
characters (this is a background enemy, must not out-pop the player). [공용 스타일 블록]
```

### 지형 타일셋 프롬프트 (수풀·벽)
```
A seamless 3x3 auto-tile set for {수풀/벽}: center tile, 4 edge tiles, 4 corner tiles,
each 112x112, designed to connect edge-to-edge with NO seams when placed on a grid.
Top-down flat (bushes) / low oblique (walls). Low saturation (background terrain).
Consistent single mass look when tiled. [공용 스타일 블록 — 단 "hero character" 문구 제외]
```

### 낱개 프롭 프롬프트 (수정·바위·문)
```
A single {오브젝트} game prop occupying {N}x{N} tiles, feet/base at bottom baseline,
provide 2-3 subtle variations. Mid saturation. [공용 스타일 블록]
```

## 5. QA 체크리스트 (재생성 후 테스트 격자에 깔고 확인)

- [ ] 전 에셋 **접지선**이 같은 높이 — 한 바닥에 서 있어 보이는가?
- [ ] 광원 방향 전부 좌상단으로 일치하는가?
- [ ] 자체 그림자·받침이 **없는가**(배경 투명)?
- [ ] 캐릭터가 몬스터·바닥보다 **채도·대비로 확실히 튀는가**?
- [ ] 수풀/벽을 ㄱ자·6칸으로 깔았을 때 **경계가 이어지는가**(seam 없음)?
- [ ] 라인 색이 검정 아닌 남보라, 굵기 일정한가?
- [ ] 파랑-온-파랑으로 파묻히는 유닛이 없는가?
- [ ] footprint대로 충돌·배치가 맞는가?

## 6. 작업 순서 (한 번에 다 뽑지 말 것)

1. **파일럿 1종 먼저** — 캐릭터 중 하나(예: 럭키)를 스킨 규격으로 재생성 → QA 통과할 때까지 프롬프트 조정. **스타일 블록을 여기서 확정.**
2. 확정된 스타일 블록으로 **캐릭터 6종** 배치 재생성 → 테스트 격자 QA.
3. **지형 타일셋**(수풀·벽) 9-슬라이스 재주문 → 격자 이음 테스트.
4. **몬스터 톤다운** + boss_king 재작.
5. 낱개 프롭 footprint·접지 정규화 + 변형.
6. 규격 확정본으로 **맵 에디터** 골격 시작 (56px 격자·footprint·y-sort 기준).
