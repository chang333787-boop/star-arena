## v1.16 — 슬라이서 v5: 분리 발사이펙트 정리 (코덱스 검토 반영) (2026-07-01)
**한 줄:** 코덱스 검토를 반영해 슬라이서를 v5로 고도화 — 공격/피격/리턴 프레임에 **떨어져 뜬 발사 이펙트(별·부메랑·종이비행기·거품)를 정확히 제거**하고, 든 무기는 유지. 디자인·게임로직·밸런스 불변.
- **`keep_main` 판정 재작성**(`tools/slice_assets.py`): bbox 겹침 → **본체와의 실제 픽셀거리 + 셀끝 도달 + 절대크기**로 판정. 본체와 사실상 맞닿은(≈3px) 든 무기(별골렘 캡슐런처 등)만 유지, **본체에서 떨어진 큰 발사 이펙트는 제거**(모아 방패 옆 분홍 별 등). 8% 상대기준 폐기로 작은 무기 오삭제 방지.
- **중심 정렬 개선**(`body_cx`): 서있는 프레임=발(하단42%) 기준, 눕는 return=본체 최대연결요소 중심 기준 → return에서 무기 때문에 본체가 안 밀림.
- **밝은 외곽선 보호**(`remove_bg` halo): 투명에 닿은 밝은 픽셀을 무조건 안 지움(순수 배경밝기+바깥잔털만) → 눈꽃 흰옷·럭키 연노랑·별골렘 하이라이트 보존, 흰 배경/하단 사각형은 계속 제거.
- 검증: 6캐릭터×5프레임 본체 픽셀 보존(캐릭터 손상 0), 모아 다리·럭키 얼굴·별골렘 형태·눈꽃 흰옷·방패 별문양 유지, 캔버스 캐릭터별 동일·경계 여백 8px, index.html↔star_arena.html SHA-256 동일, **하네스 4종 PASS**. (외부 코덱스 3회 검토 반영)

## v1.15 — 스프라이트 슬라이싱/렌더 정리 (2026-06-30)
**한 줄:** 에셋 적용 후 생긴 **공격 프레임 잘림 / 발밑 흰색(배경·그림자) 덩어리 / 프레임 간 크기·기준 불일치**를 슬라이싱·렌더 기준만 손봐 해결(디자인·이미지 재생성 없음).
- **배경 제거 방식 교체**(`tools/slice_assets.py`): 모서리 색 기준 → **밝기/회색 경계 플러드**로 셀 중앙의 흰 배경·발밑 옅은 그림자까지 투명화. 외곽선 안쪽 하이라이트는 연결성으로 보존.
- **최대 연결요소만 보존**: 흩어진 효과 조각·옆칸 침범·그림자 섬 같은 **흰/배경 찌꺼기 덩어리 제거**. 손에 든 무기는 캐릭터와 연결돼 유지(던지는 발사체는 제외 → 깔끔).
- **프레임 정렬(union box)**: 캐릭터별 idle/move/attack/hit/return **5프레임을 같은 캔버스 크기**로 저장 + **가로=발/하반신 중심**, **세로=발 기준선(하단)** 정렬 → 프레임이 바뀌어도 크기·위치가 안 튐. attack은 좌우 대칭 여백으로 무기/효과가 **안 잘림**.
- **렌더 기준 변경**(`drawCharacterSprite`): 높이 고정 + **발 기준 앵커(footY)** 로 그려 모든 프레임 발 위치 일치.
- 시뮬레이션(게임 좌표계)로 6캐릭터×5프레임 검증: 공격 안 잘림·흰 덩어리 없음·발 정렬·모아 다리/럭키 얼굴/별골렘 형태 유지. **전투/밸런스/온라인 로직 미변경**, 하네스 4종 통과.

- **고정 매칭/이름**: 럭키·달이·시고니·눈꽃·모아·별골렘 ↔ 별조각/종이비행기/반짝부메랑/비눗방울/쿠션방패/젤리캡슐. `defaultWeaponId`를 student_0N↔tool_0N으로 정렬(능력치 수치는 슬롯값 유지=밸런스 불변).

## v1.13 — 학생 계정 로그인 + 교사용 패널 (2026-06-30)
**한 줄:** 임시 닉네임 프로필 → "교사가 발급한 ID/PIN으로 로그인하는 수업용 계정 시스템"으로 재설계.
- **접속 흐름**: 타이틀(SPLASH) → **학생 로그인** → 개인 로비. (로그인 전엔 게임 진입 불가)
- **학생 로그인**: 교사가 만든 **ID + PIN**으로 로그인. 신규 계정은 Lv.1·골드0·기본 캐릭터/도구.
- **교사용 로그인**(학생 화면 아래 분리, 보라/회색 톤) → **교사용 패널**: 학생 계정 **생성/목록/PIN 재설정/초기화/삭제**.
- **개인 로비**: 상단·왼쪽에 이름/ID/Lv/골드 + **로그아웃(Q)**, 오른쪽 "내 캐릭터" 카드.
- 계정은 **별도 저장**(`starArena.accounts.v1`/`session.v1`), 기존 프로필과 충돌 없음. **PIN은 해시 저장**(원문 미저장).
- 온라인 방 player에 학생 이름/`studentId` 사용, **pinHash는 절대 전송 안 함**.
- ⚠️ 단일 HTML 기반 **수업용 간이 보호**(진짜 보안 아님). 실제 보안은 Firebase Auth+규칙 필요(README).
- 전투/온라인/Firebase 로직 미변경, 하네스 4종 통과(계정 테스트 추가).
