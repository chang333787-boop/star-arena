[작업명] BIG-BATCH-2 — 카메라 멀미 수정 + 모드 정밀분석·보충 + 온라인 탑재 + 클라우드화 + 줌아웃/대형맵

목표:
5개 Phase를 **루프로 완주**한다: 각 Phase 완료 → 커밋 → **멈추지 말고 즉시 다음 Phase**. 중간 확인 대기 없음. 진짜 블로커(중단조건)만 보고. 전부 끝나면 최종 보고 1회.
근거: 사용자 실플레이 피드백(카메라 어지러움·모드들 방향 만족·맵제작=마리오메이커 포인트·술래 캐릭 역할성) + 백로그(D10 클라우드·C6/C7 카메라·온라인 탑재).

■ 공통 금지: index.html↔star_arena.html 바이트 동일 깨기 / harness 전역 심볼 변경 / 기존 모드·캠페인·saveProfile 파손 / **사운드 제작·수급 금지**(오디오 에셋은 사용자가 학생들과 직접 제작 예정 — 훅 시스템은 이미 완성, 파일만 기다림. 신규 이벤트의 훅 등록만 OK).
■ 공통 규칙: 수치 상수화+B콘솔 · Phase마다 하니스 회귀 + 신설 검증 · 콘솔 에러 0 · Phase별 커밋.
■ 중단조건(이때만 멈춤): 줌아웃 카메라가 회귀 폭주(하니스 다수 실패 지속) / Firebase 구조 변경이 기존 온라인 방 호환을 깨는 경우 / 그 외엔 기록하고 계속.

━━━━━━━━ PHASE 0 · 플랫포머 카메라 방향전환 멀미 수정 (즉효) ━━━━━━━━
- 증상(사용자): 뒤돌면 카메라가 갑자기 전환돼 어지러움 — lookahead(+80px)가 방향 바뀌는 순간 즉시 반대로 점프하는 것.
- 수정: **lookahead 오프셋을 목표값으로 서서히 이동**(lerp, 최대 이동속도 ~200px/s 또는 0.6s ease) + **방향 유지 0.3s 후에만 lookahead 반전**(짧은 반동/뒤돌기엔 카메라 안 움직임).
- AC: 좌우 반복 이동 시 화면 급점프 0(하니스: 카메라 x 변화율 상한 검증) + 실플레이 체감.

━━━━━━━━ PHASE 1 · 모드 정밀분석 + 보충 요소 (핵심 — 분석 후 소형은 즉시 구현) ━━━━━━━━
방법론: 각 모드를 5렌즈로 진단 — ①재미 루프(긴장→해소 사이클이 있나) ②역할 다양성(캐릭 6종이 각자 "살아나는" 도구가 있나 — 사용자가 짚은 핵심: 술래에서 시고니 가시길·눈꽃 서리가 게임성을 살림) ③역전 요소(지는 팀의 희망) ④맵 상호작용(에디터 요소가 전략이 되나) ⑤대기/관전 재미(죽었을 때·기다릴 때).
산출: `모드정밀분석_결과.md` — 모드별 진단표 + 보충 제안(소/중/대) + **소형(각 모드 1~3개)은 이 Phase에서 바로 구현**, 중대형은 제안 기록.

모드별 필수 검토 항목(사용자 힌트 반영 — 이 외에도 발견하면 추가):
- **술래**: 캐릭터 역할 밸런스 확장 — 현재 시고니·눈꽃만 도구 있음. **나머지 4캐릭에도 모드 전용 "살아나는" 대체 도구 검토**(예: 럭키=짧은 가속, 별골렘=1회 잡기 방어막, 모아=주변 아군 은신 보조, 달이=먼 곳 순간 표식?) — 원본 스킬 훼손 없이 모드 한정 오버라이드. + 최후의 1인 하이라이트 연출 · 술래 수 증가 시 도망자 미세 버프.
- **칠하기**: 실측 문제 = 봇 6인이 30초에 포화 → 매치시간·칠 효율·맵 권장 크기 재조정. 종반 재미(마지막 10초 칠 2배 등) · 궁극기=대형 도색 스페셜성 검토.
- **플랫포머(마리오메이커 포인트 — 사용자 강조: 맵제작이 되면 애들이 정말 재미)**: 에디터 기믹 확장 후보 — 움직이는 플랫폼 · 부서지는 블록 · 얼음 바닥(미끄럼) · 열쇠+문 · 타임 메달(금/은/동) · 레벨 목록에 제작자명·클리어 수 표시(제작 동기!). 소형부터 즉시, 큰 건 제안.
- **수정부수기**: 벽 재건설 요소? 포탑 파괴 후 중간 목표? 진단.
- **점령전**: 거점 1개 고정의 단조로움 — 거점 이동/복수 거점 검토.
- **별모으기**: 별 많이 든 캐리어 표시(표적 긴장) · 드랍 쟁탈 재미 진단.
- **섬멸전**: 성장버프 외 차별 요소 진단.
- **캠페인·별빛 점프 연계**: 별조각/클리어 → 골드 소량 연결 검토(E15 상점 전 단계).

**1-A (필수) 에디터 저장 슬롯 5개**: 학생이 "저장됐는지" 확신 못 하는 문제(자동저장은 안 보임) → **명시적 저장 슬롯 5칸**(맵공방 공통 — 플랫포머 포함 전 모드).
- 슬롯 UI: 각 칸에 미니맵 썸네일·맵이름·모드·저장 시각. 빈 칸은 "비어 있음".
- 조작: 저장(S 또는 버튼) → 슬롯 선택 → **"저장했어요!" 명확한 확인 연출**(토스트+슬롯 반짝). 불러오기/덮어쓰기(덮어쓸 땐 확인)/삭제.
- 기존 초안 자동저장은 유지(사고 방지용 백업 — 슬롯과 별개). 저장 위치 localStorage, Phase 3 클라우드화 시 슬롯도 동기화 대상 검토.

**1-B (필수) 모드별 에디터 요소 완비 조사·구현**: 각 모드가 맵 제작에서 필요로 하는 요소가 다름 — 모드별로 ①전용 팔레트 요소가 다 있는지 ②배치→rulePoints/게임 배선이 실제 작동하는지 ③모드 특성 검증(예: 칠하기=칠 가능 면적, 술래=수풀 밀도, 수정부수기=탑 위치 대칭, 점령전=거점 중심선)이 있는지 전수 점검 → **부족분 구현**. 결과를 분석 문서에 모드×요소 매트릭스로 기록.

━━━━━━━━ PHASE 2 · 전 모드 온라인 탑재 (실측: 현재 온라인 = tdm뿐) ━━━━━━━━
- **목표: 별빛 점프 제외 전 규칙 온라인화** — 섬멸(tdm, 기존) + **점령전(hotzone)·수정부수기(siege)·별모으기(stargrab)·술래(tag)·칠하기(paint)**.
- 방식: 온라인 방 생성 시 **규칙 선택**(호스트가 고름, 방 meta에 기록 — ONLINE_MODES의 ruleType 하드코딩 해제). host 권위 유지, 빈자리 봇 동일.
- 규칙별 동기화 추가분: 점령전=게이지 · 수정부수기=탑 HP/벽 · 별모으기=별 스폰/보유/드랍 · 술래=역할 전환·감염 이벤트 · 칠하기=페인트 델타(PRD §2.6: 0.2s 배치·200칸 상한·중도입장 RLE 스냅샷).
- **별빛 점프의 온라인 = 클라우드 레벨 공유**(Phase 3) — 서로의 작품을 받아서 깨는 것이 이 장르의 멀티. 실시간 동기화 비대상(고스트는 백로그).
- AC: 규칙별 2클라이언트(호스트+게스트) 하니스 — 승패·핵심 상태(게이지/탑HP/별/감염/페인트) 일치 검증. 방 목록에 규칙 표시.

━━━━━━━━ PHASE 3 · 맵공방 클라우드화 (D10) ━━━━━━━━
- 승인맵·플랫포머 레벨·대기 목록을 Firebase(`starArenaOnline/editorMaps/<classId>`)에 저장 — **학급 전체 공유**(옆 컴퓨터·브라우저 초기화에도 유지).
- 교사 승인/삭제/비활성(D11)이 클라우드 반영 → 전 클라이언트 부팅 시 동기화. 오프라인 폴백 = 기존 localStorage(양방향 머지: 클라우드 우선).
- 개인정보 최소(맵 데이터+작성자 닉·학급 코드만). 기존 계정/학급 구조 재사용, 신규 인증 없음.
- AC: A컴퓨터 제출→B컴퓨터 교사 승인→A·B 모두 목록 반영(하니스 mock DB + 실 Firebase 스모크).

━━━━━━━━ PHASE 4 · 줌아웃 카메라 + 대형맵 플레이 탑재 (C6·C7 — 최대 리스크, 마지막) ━━━━━━━━
- **월드 좌표 ↔ 화면 좌표 분리**: 카메라 변환(scale+offset) 도입. 기본 맵(22×10) = scale 1 고정(기존과 픽셀 동일 — 회귀 0 원칙).
- **대형맵 = 고정+줌아웃**(맵 전체가 화면에 들어오는 scale로 축소, 스크롤 없음 — 멀미 회피 방침). 36×16 즉시 탑재 + 신규 48×22 프리셋(에디터에 추가).
- HUD·입력(마우스 조준 canvasToLogical·터치 스틱)·미니맵/에디터 좌표 전부 변환 경유 확인.
- 초대형 추적 카메라는 이번에 안 함(백로그 유지 — 줌아웃까지만).
- AC: 22×10 픽셀 회귀 0(스크린샷 대조) · 36×16/48×22 에디터 제작→승인→전 모드 플레이 · 유닛 가독성 확인(48×22에서 캐릭터 크기 실측 기록) · 하니스 전종 PASS.
