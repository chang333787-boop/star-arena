# 별빛 아레나 v1.13 작업 보고 — 학생 계정 로그인 + 교사용 패널

> 로그인/계정/관리자 흐름을 "게임 서비스"처럼 재설계했습니다. 전투/온라인 로직은 건드리지 않았습니다.
> 배포 기준 파일: **index.html** (star_arena.html은 동일 사본)

## 1. 기존 구조가 왜 허술했나
- "로그인"이 실제 인증이 아니라 **localStorage 단일 프로필 + 6칸 명단(비밀번호 없음)** 편집이었음.
- 관리자 모드는 6칸 이름/캐릭터만 바꾸는 수준, **계정 생성 불가**.
- 프로필이 전부 클라이언트라 학생이 골드/해금을 **임의 변경 가능**(서버 권한 없음).
- 첫 화면이 바로 설정 목록이라 "내 계정으로 접속했다"는 느낌이 없었음.

## 2. 새 접속 흐름
`앱 시작 → SPLASH(타이틀) → LOGIN(학생) → STUDENT_LOBBY → 게임/상점/온라인/내정보`
- 유효한 **학생 세션**이 있으면 SPLASH 후 바로 로비. 없으면 로그인 화면.
- **교사 세션은 자동 진입 안 함**(매번 PIN 확인).
- 로그인 전에는 게임에 들어갈 수 없음(START는 로그인 후에만 도달).

## 3. 학생 로그인 방식
- 교사가 발급한 **ID + PIN**(prompt 입력) → `studentLogin(id,pin)`.
- 성공 시: `currentStudentId` 설정 → `profile = accountToProfile(account)` → 세션 저장 → 로비.
- 신규 계정: Lv.1 / 골드 0 / 기본 캐릭터·도구(게임 기본 해금 로스터) / 전적 0 / createdAt·lastLoginAt 기록.

## 4. 교사용 로그인/패널 위치
- 학생 로그인 화면 **맨 아래 "교사용/관리자 로그인"**(보라/회색 톤, 학생 화면과 명확히 다름).
- 교사 PIN(기본 1234, `ADMIN_CONFIG`) → **교사용 패널**: 좌측 학생 목록 / 우측 선택 학생 상세 / 하단 액션.
- 액션: **A** 생성 · **P** PIN 재설정 · **R** 초기화(ID 확인) · **X** 삭제(ID 확인) · **↑↓** 선택 · **Esc** 로그아웃.

## 5. 학생 계정 생성 방식
- 패널 **A** → ID(2~12자)/이름(2~10자)/초기 PIN(4~8자) prompt 입력.
- 중복 ID 금지, 생성 직후 **"ID/PIN 학생에게 전달하세요"** 안내(이후 PIN 원문은 복구 불가, 재설정만).

## 6. PIN 저장 방식
- 원문 대신 **동기 해시(cyrb53류, `hashPin`)** 저장. 결정적·테스트 가능.
- ⚠️ 암호학적 보안 아님(수업용 간이 보호). 로그인 시 입력 PIN을 같은 해시로 비교.

## 7. 로비에서 "나 자신" 표시
- 상단 식별바: `👤 이름 · ID s01 · Lv.1 · 골드 · 온라인상태` + **Q 로그아웃**.
- 왼쪽 [내 정보] 카드: 이름/ID/레벨/골드/전적/수집.
- 오른쪽 [⭐ 내 캐릭터] 카드: 현재 선택 캐릭터/무기(라이브).

## 8. 온라인 방 ↔ 계정 연동
- 온라인 방 player에 `studentId`(+ nickname=학생 이름) 저장. Firebase 익명 uid는 동기화용으로 그대로.
- **pinHash는 방/state에 절대 포함하지 않음**(player 객체·packFighter·tPackFighter 확인).

## 9. 보안 한계(정직하게)
- 단일 HTML이라 코드/localStorage를 보면 우회 가능 → **진짜 보안 아님, 수업용 보호 수준**.
- 실제 보안은 **Firebase Auth + Realtime Database 보안 규칙** 필요.
- 계정은 기기별 localStorage(Firebase 미연동 시 여러 기기 공유 안 됨).
