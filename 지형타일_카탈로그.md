# 지형 타일 카탈로그 — batch 생성 + 맵 에디터 팔레트 설계도

> 작성: 2026-07-05 · 용도: 지형 타일을 한 번에(batch) 생성하기 위한 전체 목록 + 규칙 + 생성순서.
> 근거: `타일에셋_규격서_v0.1.md`(스타일), `맵에디터_요소확정.md`(팔레트 11종), 모드 4종 확정.
> 원칙: **파일럿 1~2장으로 스타일 잠근 뒤 나머지 batch.** (boss_king 방식)

---

## 0. 확정 맥락
- **모드 4종**: 섬멸전(있음) · 🏰수정부수기(공격 포탑) · ⭐별모으기 · ✨핫존. (🎨색칠=시즌2)
- **크기 무관 설계**: 56px 타일은 맵 크기와 독립. 큰 맵 만들어도 타일 재사용, 카탈로그 안 커짐.
- **카메라 로드맵**(별도 트랙, 지형과 무관): 고정(현재) → 고정+줌아웃(중간 큰 맵) → 부드러운 추적(초대형, 나중).

## 1. 지형 5층 구조 (맵 에디터가 층별로 다르게 다룸)
| 층 | 역할 | 렌더 | 충돌 |
|---|---|---|---|
| **L1 바닥** | 스테이지 테마 | seamless 전면 채움 | ❌ |
| **L2 벽/장애물** | 막기 | 9-slice or 낱개 | ✅ |
| **L3 엄폐** | 은신 | 9-slice | ❌ |
| **L4 함정/데칼** | 밟으면 효과 | 바닥에 눕힘 | ❌(효과만) |
| **L5 기능존** | 목표/특수 | 코드 링 or 타일 데칼 | 특수 |

## 2. 공통 규칙 (모든 지형 타일)
1. **화풍**: `타일에셋_규격서_v0.1.md`의 `[공용 스타일 블록]` — clean cel-shaded chibi, 남보라 외곽선(#2a2350), 셀 2~3단, 좌상단 광원. **지형은 저채도**(배경이라 캐릭터보다 뒤로).
2. **기능 = 색 (필수)**: 기능이 색으로 즉시 읽혀야 함. 지금 함정이 다 파래서 안 읽히는 문제 해결.
   - 위험(가시) = **빨강** · 끈적(끈끈이) = **초록/보라 젤리** · 미끌(얼음) = **하늘색** · 팀 = **청(blue)/홍(red)** · 별/목표 = **골드**
3. **seamless**(L1 바닥·L3/L2 타일셋): 큰 맵일수록 반복 격자가 잘 보임 → 엄격히 검증. 안 되면 코드 후처리(tileable화).
4. **투명배경**(L2~L5 오브젝트/데칼), 자체 그림자 금지. **L1 바닥은 불투명 전면.**

## 3. 전체 타일 목록

### L1 바닥 (테마) — seamless 256×256, 불투명
> ⚠️ **배선 주의(실측 발견)**: 현재 코드는 바닥을 256px로 렌더(index.html ~3644 `tx+=256, drawImage(...,256,256)`)하는데, 게임 격자는 **56px**(STUDENT_CELL). 256÷56=4.57 → 바닥과 격자·오브젝트가 어긋남. **새 지형 배선 시 바닥 렌더 크기를 56의 배수로**(권장 **224px**=4×56, 또는 112=2×56). 타일 그림 자체는 무관, 렌더 스텝만 수정.
| 타일 | 테마색 | 스테이지 | 상태 | 난이도 |
|---|---|---|---|---|
| 별빛 자갈 ground_basic | 파랑 | 우주/기본 | ✅ 있음 | — |
| 풀밭 grass | 초록 | 숲 | 신규 | 쉬움(seamless 검증) |
| 눈/얼음 snow | 하양파랑 | 겨울 | 신규 | 쉬움 |
| 모래 sand | 베이지 | 사막 | 신규 | 쉬움 |
| 나무 wood | 갈색 | 실내/요새 | 신규 | 쉬움 |
| 어둠 void | 진보라/검정 | 위험 | 신규 | 쉬움 |
| (밝은 중립 neutral) | 밝은회색 | 색칠전용 | **시즌2** | — |

### L2 벽/장애물 — 9-slice 336×336 or 낱개, 충돌 O, 투명
| 타일 | 색 | 상태 | 난이도 |

| 쿠션벽 cushion_wall (9-slice) | 초록 저채도 | 🔄 Codex 진행예정 | 어려움(9-slice+후처리) |

## 4. 모드 ↔ 필요 존 타일
- **섬멸전**: 없음 (아레나+벽/엄폐만)
- **🏰수정부수기**: 별 수정탑(청/홍) + 파괴/건설 벽
- **✨핫존**: 거점 데칼
- **⭐별모으기**: 중앙 별광맥 + 별 pickup

## 5. 생성 계획 (파일럿 → batch)
1. **파일럿 (스타일 잠금)** — 바닥 `풀밭` 1장(seamless 검증) + 함정 `가시`(기능색 검증). 통과할 때까지 프롬프트 조정.
2. **batch A — 바닥 나머지**: 눈·모래·나무·어둠 (파일럿 스타일 그대로).
3. **batch B — 함정**: 끈끈이·얼음.
4. **batch C — 존**: 거점 데칼 · 별광맥 · 별 pickup · 수정탑(청/홍).
5. **batch D — 벽 9-slice**: 쿠션벽(+테마 벽).
- 각 batch 후 QC 페이지에 넣어 검수 → 통과분만 채택.

## 6. 맵 데이터 legend 갱신 (에디터 연동)
기존 `맵에디터_요소확정.md` legend에 추가:
`{ ... 바닥테마 F1~F6, 거점 O, 수정탑 X, 별광맥 G, 팀스폰 청/홍 }`
- 바닥은 "칸 효과"가 아니라 "영역 페인트"라 별도 레이어로 저장(에디터 v1에서 단일 바닥 선택도 가능).
