[작업명] BIG-BATCH-1 — 시각 정리 + 캠페인 재설계 + 대형맵 + P1(멀티/UGC) + P0 잔여

목표:
진화한 게임(4모드·재설계 전투·성장버프·맵에디터·조작3모드)에 맞춰 대규모 정비. **작고 명확·고임팩트부터, 큰 건 우선순위대로.** P2(리텐션·서버권위 등)는 다음 배치.
근거: 사용자 실플레이 + `게임평가_상용화갭`(P1). **각 항목 코드로 검증 후 진행.**

■ 공통 금지: index.html↔star_arena.html 바이트 동일 깨기 / harness 전역 심볼 변경 / 캠페인·온라인·saveProfile 파손 / 대규모 단일파일 분리.
■ 공통 허용: 렌더·HUD·캠페인·에디터·온라인·밸런스·설정 수정, 새 상수/함수, 하니스/테스트/문서.
■ 진행 규칙: 아래 A(시각정리)는 작고 즉효 → **먼저**. 이후 B~F 우선순위대로. 과대하면 완료분까지 커밋+보고. **에셋 필요/설계결정 필요 항목은 구현 대신 "권고+대기"로 기록**(사운드 에셋·캠페인 세부 등).
■ 중단조건: 조작3모드(UPDATE-1)와 미머지 충돌 / 온라인 동기화 위험 / 회귀로 캠페인 클리어 불가.

━━━━━━━━━━━━━━━━ A · 시각 정리 (원인 분석됨 — 먼저, 즉효) ━━━━━━━━━━━━━━━━
1. **벽 '표처럼' 수정** — 벽이 56px 셀 개별 오브젝트(~2217)로 컴파일되어 각 셀이 완전 9-slice(사방 테두리, `drawWallTile` ~4431)로 그려짐 → 표 꼴. **이웃 인식 오토타일로 전환**(수풀 `drawBushesAutotile` 방식 — 내부 셀=센터조각 무테, 외곽만 테두리) 또는 연결 벽 영역 병합 후 1개 9-slice. 충돌은 셀 유지, 렌더만 병합.
2. **수풀 초록 영역 테두리 제거** — `drawBushesAutotile`의 `rgba(46,150,86,0.30)` roundRect(은신영역 표시, ~4451) 제거 또는 거의 투명(≤0.06)으로. 수풀 그림만으로 식별 충분.
3. **격자선 플레이 중 숨김** — `drawArenaFloor` 격자선을 플레이 화면에선 없애거나 극히 투명(에디터에선 유지). (64/56 불일치도 자연 해소; UPDATE-1과 겹치면 56 통일 우선.)
4. **HUD 스킬명 텍스트 제거** — 스킬독 위 이름 글자 라벨 삭제(아이콘 독으로 충분). 키캡·상태링만 유지.

━━━━━━━━━━━━━━━━ B · 캠페인 재설계 ━━━━━━━━━━━━━━━━
5. 현재 캠페인(20스테이지 PVE)을 **진화한 게임에 맞게 재설계.** 방향(제안 — 세부는 사용자 확정 필요, 큰 결정은 기록):
   - **성장버프(처치→버프) 캠페인 통합**, **바닥 테마별 구역**(풀/눈/사막/나무/어둠 = 세계관 진행감), **새 보스 4종 활용** 보스전 강화.
   - 난이도 곡선 재조정(재설계 전투 기준), 강화 3택1 유지·확장.
   - (선택) 일부 스테이지에 오브젝트 변형(거점 방어 등) — 단조로움 완화.
   - ※ 캠페인 서사/스테이지 구성 세부는 **사용자와 별도 설계**(여기선 골격+통합만, 확정 전 대개편 금지).

━━━━━━━━━━━━━━━━ C · 대형 맵 + 에디터 ━━━━━━━━━━━━━━━━
6. **아주 큰 맵 크기 신설** — 22×10·36×16 외 더 큰 규격(예 48×22) 구상, **에디터 크기 선택지에 추가**.
7. **대형맵 플레이 탑재** — ARENA 고정(1232×576) 해제, **줌아웃/추적 카메라**로 화면보다 큰 맵 플레이. (멀미 회피: 고정+줌아웃 우선, 초대형만 부드러운 추적.)
8. **에디터 개선** — 제출 전 **테스트플레이**, **undo**, **드래그 페인팅/사각형 도구**, **초안 자동저장**(Esc로 소실 방지).
9. **에디터 마우스 입력** — UPDATE-1 조작3모드와 연계해 마우스로 배치(키보드 유지). 한 칸씩 배치 고행 해소.
