## 0. 한 줄 요약
학생들이 플레이하며 요청한 게임 기능 13가지를 추가했습니다. 로컬 / 온라인 1v1 / 3v3 / PVE / 에셋 교체 구조를 깨지 않았고, 자동 테스트(하네스) 4종이 모두 통과합니다. 밸런스 수치는 바꾸지 않았습니다(보스만 커진 만큼 소폭 보정).

## 1. 작업 방식 (안전 절차)
1. 작업 전 4개 하네스 PASS 확인 → 작업 브랜치 `feature/student-feedback-v111` 생성
2. 코드 구조를 먼저 조사(읽기 전용) 후, **작은 묶음(A~F)**으로 나눠 구현
3. **각 묶음마다 하네스 4종을 돌려 통과할 때만 커밋**
4. 마지막에 **다중 에이전트 적대적 검수**로 버그를 잡아 수정 → main 머지 → 배포 → 라이브 확인

## 3. 핵심 설계 원칙(검수 포인트)
- **host 권한**: 온라인에서 탄창/장전/궁극기 게이지/함정/아이템은 **모두 host가 판정**. 게스트는 자기 입력만 전송(`reload`·`super` 포함), 우회 불가.
- **전송량 유지**: state에 이미지/큰 데이터 없음. 추가는 작은 숫자(`am`,`rl`,`sg`)와 `powerups` 배열뿐. 정적값(charId/team/maxHp, 적 color/r)은 게스트가 로스터/타입으로 역산.
- **밸런스 불변**: 캐릭터/무기 능력치 mul은 그대로. 능력치 막대는 그 값을 "시각화"만. (보스만 크기에 맞춰 속도/연사 소폭 완화)
- **장전과 궁극기는 별개**: 궁극기는 탄창과 무관하게 게이지만 차면 사용(요청서 명시).
- **PVE는 스테이지 맵 유지**: 온라인 맵 선택은 1v1/3v3에만 적용, PVE는 기존 PVE_STAGES 흐름 존중.

## 4. 적대적 검수에서 잡아 고친 것
- **[치명]** `readLocalInput`/`emptyInput`에 `reload` 필드가 빠져 **온라인 R 장전이 실제로 안 되던 버그** → 필드 추가 + 회귀 테스트 추가.
- 로컬 AI·온라인 봇 이동이 함정 둔화/버프를 무시하던 불일치 → `effSpeed` 적용 + 봇 상태 tick 추가.
- 온라인 메뉴/로비 배경 맵을 선택 맵과 일치, powerupId 리셋 누락 정리.
- (비버그로 판정해 변경 안 함: 궁극기가 탄창 무시 — 설계 의도.)
