# 코덱스에 전달하기 — 체크리스트 & 첫 메시지

## 1. 코덱스에게 줄 파일 (repo에 이미 다 있음)

VSCode 코덱스는 같은 repo 안에 있으니 **따로 첨부할 필요 없이 경로만 알려주면 됨.** 아래가 읽어야 할 파일:

| 순위 | 파일 | 역할 |
|---|---|---|
| ★ | `코덱스_핸드오프_브리핑.md` | **먼저 통째로 읽을 온보딩+로드맵** |
| ★ | `타일에셋_규격서_v0.1.md` | 이번 작업의 단일 기준(규격) |
|  | `에셋_분류표.html` | 현재 적용 에셋 시각 목록 |
|  | `reference/02_basic_motion_master.png` | 캐릭터 마스터 시트 |
|  | `reference/01_fixed_character_weapon_master.png` | 캐릭터+무기 마스터 |
|  | `reference/03_development_reference_sheet.png` | 캐릭터 개발 파이프라인 |
|  | `tools/slice_assets.py` | 마스터 시트 슬라이서 v6 |
|  | `index.html` | 에셋 로더(≈L548~612)·격자(L428,L659)·몬스터(L5089) |

## 2. 코덱스에 붙여넣을 첫 메시지 (이거 그대로 복붙)

```
이 프로젝트(별빛 아레나) 처음 보는 거야. 먼저 repo 루트의
`코덱스_핸드오프_브리핑.md` 를 통째로 읽고, 거기서 지정한 참고 파일
(`타일에셋_규격서_v0.1.md`, reference/ 시트 3장, tools/slice_assets.py,
index.html 에셋 로더 부분)까지 확인해줘.

목표는 맵 에디터를 만들기 전에 에셋을 하나의 규격으로 정비하는 거야.
너는 지금 Phase 0 (현황 감사 + QA 하니스 + 슬라이서 일반화 계획)만 하면 돼.
그림 생성은 네가 하지 말고(사람이 ChatGPT에서 시트를 넣어줌), 슬라이스·배선·
에디터·QA만 담당해.

Phase 0 다 하면 감사 리포트랑 하니스 결과부터 보고하고, 그 다음은 내 확인
받고 진행해.
```

## 3. 주의점 (사람이 기억할 것)

- **그림 생성은 코덱스가 못 함.** 몬스터·타일 새 그림은 네가 ChatGPT 앱에서 **월드 마스터 시트**로 뽑아 repo에 넣어줘야 Phase 1이 굴러감.
- 코덱스가 그림을 임의 생성하거나 API를 붙이려 하면 막을 것.
- 캐릭터/마스터 시트는 승인 없이 변경 금지.
- `index.html`/`star_arena.html`은 바이트 동일 유지.
