# 🌙 별빛 마을 RPG 위키 — 안내서 (README)

| 항목 | 내용 |
|---|---|
| 문서 | RPG 위키 총괄 안내 (RPG-0 산출물) |
| 단일 출처 | `RPG모드_PRD.md` v1.0.1 — 특히 §7(데이터)·§8(위키·교육) |
| 상태 | 초안 → **교사 확인 후 승인** (RPG-0 게이트) |
| 대상 | 교사(운영 규칙 전체) · 학생(별표 ⭐ 붙은 절 낭독용) |

이 폴더(`wiki/`)는 우리 반이 만드는 **별빛 마을 세계의 정본(定本)**입니다.
학생이 종이 시트에 그린 몬스터·도구·작물이 여기(위키)를 거쳐 게임 속 '별의 도감'에 실립니다.

---

## 1. 위키 3층 구조 ⭐

| 층 | 어디에 | 무엇이 | 누가 만지나 |
|---|---|---|---|
| **1층 실물** | 교실 게시판 + 종이 시트(1인 1장) | 아이디어 원본 — 그림·⭐색칠·털실 연결선 | 학생 (실명 OK — 교실 안에서만) |
| **2층 정본** | 이 폴더 `wiki/` | 항목 md 문서 + `rpg_db.json` + 변환표 | 교사(전사·승인) — 공개 산출물은 **별명/이니셜만** |
| **3층 무대** | 게임 속 **'별의 도감'**(피코의 관찰 노트) | 승인된 콘텐츠 카드 + 만든 사람 크레딧 | 게임이 자동 표시 |

- 세 층을 **하나의 ID**(예: `M-07`)가 관통합니다 — 시트 번호 = 게시판 카드 번호 = 위키 문서 번호 = json id = 도감 번호. ("내 번호가 게임에 있다"는 효능감의 핵심 장치)
- 도감 v1은 **3탭(몬스터/도구/작물)**, 8탭 완성은 RPG-5.
- 검토 중인 항목은 도감에 실루엣으로 표시: **"??? 연구 중"**.
- **빈 도감 번호는 일부러 남깁니다** — "다음 설계는 네 차례"라는 유인. 전입생·결석생에게 우선 배정합니다(§8.5).

## 2. 폴더 규칙

```
wiki/
├─ README.md                ← 이 문서 (총괄 안내)
├─ 00_세계관.md              ← 챕터 1 세계관 정본 (학생 낭독 가능)
├─ 변환표_밸런스.md           ← ⭐→숫자 변환표 + 전수 검산 결과 (교사·구현용)
├─ database.rules.초안.md    ← RPG-5 클라우드 규칙 확장 초안 (구현용)
├─ tools/
│   └─ verify_conversion.py ← 변환표 전수 검산 스크립트 (python3)
├─ rpg_db.json              ← (예정) 챕터 1 기본 데이터 정본 — RPG-0 후속
├─ sheets/                  ← (예정) 시트 8종 인쇄 양식 — RPG-0 후속
├─ monsters/  M-XX_이름.md   ← (예정) 학생 항목 문서 — 차시 2부터
├─ weapons/   W-XX_이름.md      · 도구(장비)  — 차시 3부터
├─ crops/     C-XX_이름.md      · 작물        — 차시 4부터
├─ npcs/      N-XX_이름.md      · 주민        — 차시 4부터
├─ quests/    Q-XX_이름.md      · 퀘스트      — 차시 5부터
└─ maps/      L-XX_이름.md      · 지역(챕터1은 종이 설계만 — §0.3)
```

- 항목 문서 파일명: `<ID>_<한글이름>.md` (예: `monsters/M-07_뿔깡총.md`).
- 항목 문서 머리에는 공통 헤더를 적습니다: **ID · 이름 · 만든 사람(별명/이니셜) · 상태 · 연결(🔗)**.
- 폴더는 필요해지는 차시에 만듭니다(빈 폴더 커밋 금지). 커밋·push는 **메인 세션(교사)** 담당.

## 3. ID 체계 ⭐

| 접두어 | 분류 | DB 테이블(내부명) | 예 |
|---|---|---|---|
| **M** | 몬스터 | monsters | M-01 그림자 젤리 |
| **W** | 도구 | weapons(노출 명칭은 '도구' — §0.4 순화) | W-02 호미 |
| **A** | 옷·장신구 | armors | A-01 천 조끼 |
| **I** | 재료·아이템 | items | I-01 젤리 조각 |
| **S** | 스킬 | skills | S-01 빙글 베기 |
| **J** | 직업 | jobs | J-01 별빛 검사 |
| **C** | 작물 | crops | C-01 별무 |
| **N** | 주민(NPC) | npcs | N-01 촌장 온별 |
| **Q** | 퀘스트 | quests | Q-01 별마중 마을 도착 |
| **L** | 지역(맵) | maps | L-01 별마중 마을 |

- 번호는 **두 자리(01~99), 제출 순서대로** 발급하고 **한 번 주면 바꾸지 않습니다**(결번도 그대로 — 빈 번호는 다음 설계 유인).
- **기본 콘텐츠가 앞 번호를 선점**합니다: 몬스터 M-01~M-08(보스 포함), 도구 W-01~W-07, 작물 C-01~C-06 등. 학생 콘텐츠는 그 다음 번호부터.
- 위키 ID와 **게임 DB id는 별개**입니다: 학생 콘텐츠의 DB id는 `stu_<type>_<n>` (예: M-12 → `stu_monster_12`)이며 **제출 시점에 영구 확정** — T3 승격 후에도 불변, 개명이 필요하면 구→신 alias 맵으로 마이그레이션(§7.1). 기본 콘텐츠는 가독 id(`mq01`, `rpg_village` 등). 항목 문서 헤더에 두 id를 병기합니다.

## 4. 상태 흐름 ⭐ (시트의 '상태 도장'과 동일)

```
📝 초안 ──→ 🔍 검토 ──→ ✅ 승인 ──→ 🎮 반영
(종이 시트)  (pending·      (approved·    (RPG_DB 병합·
             자동 검증기)    교사 패널)     도감 카드+크레딧)
                │
                └─→ 🌱 온실(보류) — 탈락이 아니라 "아이디어 온실"에 보관, 언제든 복귀
```

| 상태 | 있는 곳 | 규칙 |
|---|---|---|
| 📝 초안 | 종이 시트·게시판 | 숫자 금지 — ⭐색칠과 보기 동그라미만 |
| 🔍 검토 | `pending` (T2 스토어) | 자동 검증기: 범위 클램프 + 버짓 초과 시 비율 스케일다운 + 조정 안내("공격력 9999→13로 조정했어요") — **반려 없음**(§7.4) |
| ✅ 승인 | `approved` | 교사 승인 패널이 최종 게이트(스탯 막대+버짓 게이지 미리보기·실명 검출 경고) |
| 🎮 반영 | `RPG_DB` (T1/T3) | 부팅 병합 `merge(T1, approved)` · 내리더라도 삭제 아닌 `disabled` 비파괴 숨김 |

- 등록 3계층(§7.2): **T1** 내장 `const RPG_DB` → **T2** 로컬 스토어+클라우드 미러(RPG-5) → **T3** 교사 JSON 내보내기→내장 승격→커밋·배포. T3 절차에는 star_arena.html 미러 동기(cp)가 포함되며 전부 **메인 세션 소관**.
- **전사(시트→JSON)는 차시별 롤링**이 원칙 — 7차시 뒤 일괄 전사 금지(학급 100장+ 시 교사 5~10시간). RPG-4의 배치 intake 도구(사진 일괄→AI 변환→오류 목록)를 사용.

## 5. 시트 8종 공통 원칙 (PRD §8.2 요약)

1. 공통 헤더: **ID · 이름 · 만든 사람 · 상태 도장** 칸.
2. **숫자 금지** — ⭐색칠과 보기 동그라미만. 숫자는 변환표(`변환표_밸런스.md`)가 자동으로 정합니다.
3. **⭐예산제** — 별을 다 가질 수는 없어요(트레이드오프 교육 + 밸런스 1차 방어). 규칙을 지킨 시트는 **자동 검증을 무조건 통과**하도록 역산 설계됨(§7.5 — 전수 검산 완료).
4. **🔗연결 칸 필수** — 저학년 1개, 고학년 3개. (예: "내 몬스터가 떨어뜨리는 재료 → 친구의 도구 재료")
5. 그림 칸 — "배경은 그리지 않아요."
6. 퀘스트 시트의 대사 2종은 **빈칸 채우기 템플릿**, nextHint는 선택(작성 25분 상한).
7. 지역 시트는 **24×12 테두리 + legend 사전 인쇄** 양식(챕터 1은 종이 설계 + 챕터 2 예고).

## 6. 8차시 수업 흐름 한눈에 (PRD §8.3)

| 차시 | 주제 | 산출물 |
|---|---|---|
| 1 | 세계관 탐험 | 개인 플레이 + '열린 여백' 선언 |
| 2 | 몬스터 | M카드 20~30장(연결 원료) |
| 3 | 장비(도구) | **친구 몬스터의 재료로만** 제작(연결 강제) |
| 4 | NPC·작물 | 마을 배치도 |
| 5 | 퀘스트 | 털실로 카드 3개 잇기 |
| 6 | 지역 | 모둠 종이 설계(게임 반영은 챕터 2 예고) |
| 7 | 편집 회의 | 체크리스트 상호검토 + 스티커 투표 |
| 8 | **업데이트의 날** 🎉 | 패치노트 낭독 + 자기 콘텐츠로 플레이 |

- 각 차시 도입 10분은 개인 플레이 트랙(밭 관리·퀘스트) — 작물 연속성 확보.
- P2 학급 개방(RPG-5) 전 P1 기간에는 도입 10분을 공용 월드 시연+취침 교대로 대체.
- 조기 완성자는 '연결 미션 카드'.

## 7. 거버넌스 — 우리 반의 약속 ⭐ (PRD §8.4)

- **위키 수록은 관대하게, 게임 반영은 엄격하게.** 탈락은 없다 — 보류는 '아이디어 온실'.
- **원안 보존, 숫자만 조정.** 조정 내역은 패치노트에 투명 공개(그 자체가 밸런스 수업 소재).
- 이름은 **한글 2~8자**. 실존 인물·급우 실명·기존 주인공 6명 이름은 금지(검증기 금칙 목록).
- **아무도 죽지 않는다** — 쓰러지면 별가루가 되어 하늘로 돌아간다. "1학년 동생 기준"으로 쓴다.
- 친구 항목을 나쁘게 쓰려면(약점 재료 등) **친구 동의 필수**.
- **개인정보 기본값**: 공개 산출물(T3 커밋·클라우드 미러)은 **별명/이니셜 강제**. 실명은 교실 실물 게시판까지만.
- 크레딧: 도감 카드 하단 상시 + 패치노트 호명 + 칭호(3개 견습 기록가 / 7개 도감 연구원 / 채택 개척자).

## 8. 운영 수명주기 (PRD §8.5)

- **학년 교체**: 연도별 아카이브 — 전년도 학생 콘텐츠는 disabled 일괄 + 도감 **'명예의 전당'** 탭으로 이동(T1 영구 내장분 포함). 새 학급은 빈 위키 + 명예의 전당 열람으로 시작(계승/초기화는 교사 선택).
- **전입생·결석생**: 열린 여백(00_세계관.md §여백)·빈 도감 번호를 우선 배정하는 보충 미션 — "네가 채울 자리가 남아 있어."
