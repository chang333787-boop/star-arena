# Asset Slicing Fix Note

캐릭터가 공격 모션에서 잘리거나, 배경 찌꺼기가 남거나, 프레임 전환 시 위치·크기가 튀는 문제를 해결하기 위한 에셋 재슬라이싱 노트입니다.

## 이력
### v5 회차3 (2026-07-01) — 현재: 모아 attack 분리 별 제거
- 코덱스 재검토로 확인된 마지막 1건. 회차2의 "방패와 solid 융합" 판단은 **오류**(별은 별개 알파 덩어리, 본체와 ~11px 떨어짐).
- 수정: `keep_main`의 **`D_NEAR` 16 → 6**. 좌표 절단 없이 본체와 사실상 맞닿은(≈터치) 별개 덩어리만 유지. 전체 30프레임 조사 결과 진짜 별개 무기는 별골렘 런처(dist 2.8~3.0px)뿐이라 안전.
- 결과: 모아 attack 별 제거(쿠션 방패·내부 별 문양·다리·손 보존), 덤으로 stray 효과(럭키/눈꽃 return 거품, 시고니/모아 hit) 정리. **본체 픽셀 30/30 동일**(캐릭터 손상 0), 별골렘 런처 유지, 경계 여백 8px, HTML SHA-256 동일, 하네스 4종 PASS.

### v5 (코덱스 검토 반영, 2026-07-01)
- `keep_main()` 판정을 **본체 거리장(D_NEAR=16) + 셀끝 도달(EDGE_MARGIN) + 절대크기(MIN_WEAPON=250)** 로 재작성. bbox 겹침·8% 상대기준 폐기 → 든 무기(방패·런처)만 유지, 떨어진 별·부메랑·종이비행기·거품·타격효과·그림자 섬 제거. 럭키 attack 덩어리 2→1.
- **2단 임계값**(판정 alpha>115 / 가장자리 복원 45~115)으로 반투명 잔다리로 무기에 붙은 효과 분리.
- `body_cx()`: 서있는 프레임=본체 하단42%(발), 눕는 return=본체(최대 연결요소) 중심 → return에서 부메랑 등으로 본체 안 밀림.
- `remove_bg()` halo: 'min≥222 & 투명이웃 2개 이상 바깥 잔털'만 제거로 완화 → 눈꽃 흰옷/장식·럭키 연노랑·별골렘 하이라이트 보존(흰픽셀 1499→1499).
- 검증: 캔버스 6/6 동일, 경계여백 8px, 별골렘 하단 흰 0~17px, HTML SHA-256 동일, 하네스 4종 PASS.
- 남은 연결형 이펙트: 럭키 총구 머즐플래시(총에 연속·별조각 발사기라 자연스러움), 모아 쿠션방패 옆 별(원본 아트 solid 융합, 안전 분리 불가 → 유지).

### v4 (256×256 고정) — 문제 확인됨
- 모든 프레임을 `256x256` 고정 캔버스에 **콘텐츠 전체 bbox(캐릭터+효과) 중앙 정렬 + 하단 정렬**로 저장.
- 검수에서 발견된 문제(모두 에셋/슬라이서 문제, 게임 로직 아님):
  - **A. 공격 시 본체 위치 튐**: 효과까지 포함한 bbox를 중앙에 맞춰 캐릭터 본체가 왼쪽으로 밀림(화면상 15~21px, 시고니 최대 20.6px).
  - **B. 효과 분리·공중부양**: 부메랑/종이비행기/거품/별이 캐릭터와 떨어져 조각(최대 5개)으로 떠 보임.
  - **C. 작게 보임**: 256 캔버스에서 캐릭터가 세로 32~47%만 차지(여백 과다) → 화면 24~36px로 작게 렌더.
- ※ 투명화·발(하단) 정렬 자체는 정상이었음(발 y편차 0~2px, 흰 사각 없음).

### 현재(안정형) — 위 A·B·C 해결
- `tools/slice_assets.py`를 **본체 기준 정렬 + 최대 연결요소 보존 + 캐릭터별 최소 캔버스(union box)** 방식으로 교체.
  - 배경 제거: 밝기/회색 경계 플러드(중앙 흰배경·발밑 옅은 그림자까지) + 외곽선 안쪽 하이라이트 보존.
  - **최대 연결요소만 보존** → 떨어진 발사체/그림자 섬/옆칸 침범 제거(손에 든 무기는 연결돼 유지, 던지는 발사체는 게임 자체 탄환이 그림).
  - **가로 = 발/하반신 중심**, **세로 = 발 기준선(하단)**, 캐릭터별 5프레임 **같은 캔버스 크기**로 저장 → 프레임 전환 시 크기·위치 안 튐, 공격도 좌우 대칭 여백이라 **안 잘림**.
- 검증(게임 좌표계 시뮬레이션, r=24 기준): 공격 본체 이동 15~21px→대부분 <8px(달이 0.5·모아 3.6), 분리 조각 최대 5→1, 화면 높이 24~36px→64~66px. 발 정렬·흰 찌꺼기 없음·모아 다리/럭키 얼굴/별골렘 형태 유지.
- `ASSETS_ENABLED=true`, index.html ↔ star_arena.html 바이트 동일 유지. **게임/밸런스/온라인/PVE 로직·캐릭터 디자인·고정 매칭 미변경.** 하네스 4종 PASS(이 폴더 HTML로 실행).
